# NLP 数据增强

## 是什么

NLP 数据增强是指在有限的文本数据集上通过变换生成新的合成样本，以扩充训练数据、提升模型泛化能力的技术。与图像领域（翻转、旋转、裁剪）的成熟增强方法不同，NLP 的数据增强面临根本性挑战——图像增强不改变语义（翻转一只猫仍然是猫），但文本的微小扰动可能彻底改变含义（"好看"变成"不好看"）。因此 NLP 增强需要在"多样性"和"语义保持"之间谨慎权衡。

常用方法分为三类：

1. **词汇级增强**：在词层面替换或删除，计算成本低但语义保持能力有限。包括同义词替换（SR）、随机插入（RI）、随机交换（RS）、随机删除（RD）。
2. **句子级增强**：重写或翻译整个句子，语义保持好但计算成本高。包括回译（back-translation）、句子改写（paraphrasing）。
3. **混合级增强**：在 Embedding 空间做操作，不改变离散文本。包括 [[embedding-perturbation]]（FGM/PGD）和 Mixup。

| 方法 | 成本 | 语义保持 | 适用数据量 | 实现难度 |
|------|------|----------|-----------|---------|
| 同义词替换 | 低 | 中 | 少/中 | 低 |
| 随机删除 | 极低 | 低 | 少 | 极低 |
| 回译 | 高 | 高 | 少 | 中 |
| 句子改写 | 高 | 高 | 少 | 高 |
| 对抗扰动 | 中 | 高 | 少/中 | 中 |

## 为什么重要

NLP 标注数据的获取成本远高于图像——标注一篇法律文书可能需要专家数小时，而标注一张猫狗照片只需要数秒。数据增强在以下场景中尤为关键：

1. **数据量不足**（几百到几千条）：模型容易过拟合，增强可以显著提升泛化能力。同义词替换 + 随机删除的组合在小数据集上通常能带来 2~5 个百分点的提升。
2. **类别不均衡**：对少数类使用更强的增强策略（如回译）来平衡分布。
3. **领域适应**：将通用领域的数据增强到特定领域（如将新闻文本改写为医学文本）。
4. **与正则化协同**：数据增强是隐式的 [[../regularization/regularization]]，与 weight decay、dropout 等显式正则化方法互补。

## 如何使用

### 同义词替换（SR）

最简单、最常用的词汇级增强。使用 WordNet、同义词词典或预训练 Embedding 查找近义词。

```python
import nltk
from nltk.corpus import wordnet

def synonym_replacement(text, n=3):
    words = text.split()
    new_words = words.copy()
    replaceable = [i for i, w in enumerate(words)
                   if wordnet.synsets(w)]
    n_replace = min(n, len(replaceable))
    selected = random.sample(replaceable, n_replace)
    for i in selected:
        synonyms = wordnet.synsets(words[i])
        if synonyms:
            lemma = random.choice(synonyms).lemmas()[0].name()
            new_words[i] = lemma
    return ' '.join(new_words)
```

参数 `n` 通常设置为句子长度的 10%~20%。替换太多会改变语义。

### 随机删除（RD）

每个词以概率 `p` 被删除。一般设为 `p=0.1`。实现极简，在短文本上效果不错，但长文本上过度删除会破坏语义。

### 回译（Back-translation）

将文本翻译到中间语言再翻译回原文。例如：中文 -> 英文 -> 中文。回译能产生语义高度一致但措辞不同的变体。

```python
# 思路示意：使用翻译 API 或模型

```

常见中间语言：英文（如果是中文原文本）、德文、法文。不同的中间语言产生不同类型的变化。回译质量很大程度上取决于中间翻译模型的质量。

### 不同场景的选择策略

| 场景 | 推荐方法 | 理由 |
|------|---------|------|
| 文本分类（情感/主题） | SR + RD + 回译 | 增强多样性，语义保持要求不高 |
| 文本匹配/语义相似度 | 回译 | 需要精确保持语义 |
| NER/序列标注 | SR（限制实体替换） | 不能改变实体边界 |
| 文本生成 | 回译、句子改写 | 需要保持流利度和语义 |
| 低资源语言 | 回译 + 对抗训练 | 有限数据下最大化收益 |

### 规模控制

增强数据并非越多越好。经验规则：
- 数据量 < 1000：扩增 3~5 倍
- 数据量 1000~10000：扩增 1~2 倍
- 数据量 > 10000：只对困难样本或少数类做增强
- 始终保留原始数据，防止增强数据稀释信号

## 常见误区

1. **不分场景盲目使用增强**：某些任务（如语义等价判断）对语义极其敏感，同义词替换可能改变前提关系。例如替换"没关"为"关闭"——"门没关"变为"门关闭"，语义截然不同。
2. **增强强度过大**：替换 50% 的词看似能大幅扩增数据，但产出的样本可能完全不反映真实数据分布。在替换/删除时保持保守（10%~20%）。
3. **忽视标签一致性**：回译通常保持标签不变，但同义词替换在某些任务中可能翻转标签（如将"极差"替换为"极好"）。需要验证增强后的样本标签是否正确。
4. **只做一个 epoch 的增强**：更强的做法是在每个 epoch 重新生成增强数据（on-the-fly augmentation）。这提供更多的多样性而非固定的扩增数据集。
5. **在验证/测试时也做增强**：数据增强只应在训练阶段使用。验证和测试必须使用原始未修改的数据。
