# 负采样

## 是什么

负采样（Negative Sampling）是在推荐系统各阶段（召回、粗排、精排）训练中，从海量候选物料中挑选充当负样本的策略。由于推荐场景的正样本（用户点击/转化）极其稀疏，如何构造高质量负样本直接决定了模型能否学到有效的判别边界。

不同阶段对负采样的要求不同：

- **召回阶段**：候选集是全量物料（百万到亿级），需要让模型学会区分"用户感兴趣的"和"用户不感兴趣的"。
- **粗排阶段**：候选集是召回结果（千到万级），需要让模型在较难的样本中做筛选。
- **精排阶段**：候选集是粗排结果（百到千级），需要极高的判别精度。

## 为什么重要

正样本是天然确定的（用户真实点击/转化），而负样本的选择空间极大——**你是如何选择"不感兴趣"的，决定了模型学到的决策边界在哪里**。

- 如果负样本太简单（如随机抽取全量物料中的样本），模型不费吹灰之力就能区分正负，但实际线上遇到的负样本往往是"看起来很像但不合适"的困难样本，导致模型判别能力不足。
- 如果负样本太难（全部选与正样本极相似的样本），模型可能学到过于激进的判别边界，误杀大量潜在正样本。
- 负采样不当还会引入偏差（Selection Bias），使模型学到的分布偏离真实 Serving 分布。

实践中最常见的灾难级错误：**负采样时误将正样本混入负样本集**，此时模型看似正常训练但实际上在学错误信号，表面 AUC 可能不降但线上效果奇差。

### 负采样为何比曝光未点击更优

直觉上看，"曝光未点击"似乎就是"用户不感兴趣"的直接证据。但在推荐系统中，曝光未点击样本的分布已经经过前序模型的筛选（曝光本身就是推荐系统决策的结果），因此存在严重的 Selection Bias：

- 只有被推荐系统认为"可能感兴趣"的物料才会被曝光给用户。
- 用户不点击可能是因为没注意到（Position Bias）、已经看过（Novelty）、或者当时不方便（Context Bias），而不是真正不感兴趣。
- 用这样的样本做召回负样本，召回模型学到的其实是"纠正精排的过拟合"，而不是"理解用户偏好"。

全局随机负采样则避免了这些偏置：在随机采样中，模型被迫区分"用户真正喜欢的"和"绝大多数用户不喜欢的"，学到的判别边界更鲁棒。

### 困难负样本挖掘（Hard Negative Mining）

在召回和粗排中，一种极为有效的改进是在随机负采样基础上加入困难负样本：

- **Batch 内负采样**：同一 batch 中其他 user 的正样本作为当前 user 的负样本。因为在 batch 内样本同属一个 mini-batch 分布，难度适中且计算零成本。
- **模型动态采样**：用当前模型对候选集打分，选取分数接近正样本的物料（即"模型容易误判为正"的样本）作为困难负样本。这种方式计算量大但效果显著，尤其在召回模型的后期迭代中。
- **混合比例**：建议困难负样本占总负样本的 10%~30%，太低则效果不明显，太高会使训练不稳定。

## 如何使用

### 召回阶段的负采样

**全局随机负采样 > 曝光未点击负样本**。召回阶段正样本极度稀疏（点击率通常 < 1%），曝光未点击样本本质上是经过粗排/精排筛选后的结果，分布已经严重偏倚，用它做负样本会导致召回模型只学会"不推荐精排筛掉的物料"，而不会区分"用户到底对什么感兴趣"。全局随机负采样更接近召回的真实 Serving 分布（全量候选）。

具体做法：
- 每个正样本搭配 10~1000 个随机负样本（取决于业务规模和计算资源）。
- 可以引入"简单负样本 + 困难负样本"混合：大部分随机负 + 少量曝光未点击负，既可保证判别边界不过于宽松，又不至于太困难。
- 务必定期检查负采样结果中是否混入正样本——这在数据流异常时极其常见。

### 粗排阶段的负采样

**用精排结果的 Top-K 做负样本**。粗排的候选来源于召回，正样本是用户最终点击/转化的物料。粗排的负样本如果仅用召回未命中精排的物料，难度过低。实践中有效且稳定的做法：

- 将精排模型对候选物料打分排序。
- 取排序靠后的 K 个样本作为粗排的负样本。
- 这些样本"经过了召回+粗排两关筛选"但最终被精排判定为低分，与粗排面临的真正判别场景高度一致。
- 经验表明这一策略在绝大多数推荐场景有正向效果。

### 精排阶段的负采样

精排负样本通常来自曝光未点击样本，因为这些样本已经经过了前两阶段的筛选，与正样本的分布最接近。也可以引入一些 harder negative（如精排分数接近但未点击的样本）。

### 采样比例与动态调整

负采样比例本身就是一个超参数。建议从 1:1 开始，逐步拉大负采样比例观察 AUC/NDCG 变化。负采样比例过大会引入太多噪声，过小则判别力不足。数据量越大，负采样比例可以越高（更大的海量样本中负样本的质量和信息量也更高）。

### 曝光未点击负样本的正确使用场景

虽然曝光未点击样本不适用于召回阶段，但在精排阶段反而是最佳选择：

- 精排的候选集已经是经过召回+粗排两级筛选的"高潜力物料"，曝光未点击样本与正样本在难度上最接近。
- 曝光未点击样本天然包含了系统的 Position Bias 信息，精排模型需要学会消除这种偏差。
- 精排阶段不需要全局随机负采样——那会引入太多"明显不相关"的简单负样本，使模型在精排场景下判别力不足。

关键认知：**召回需要全局分布，精排需要局部判别。** 负采样策略应该随着漏斗向下游移动而从"宽"变为"严"。

### 负采样中的偏差校正

当负采样是非均匀的（几乎所有实用的负采样策略都是非均匀的），训练分布与真实 Serving 分布之间就存在偏差。严格的做法是做偏差校正：

- **Importance Sampling 校正**：每个负样本的 Loss 按采样概率的倒数加权，`loss_weight = 1 / p(sampled)`。这样即使采样非均匀，期望 Loss 仍然是无偏的。
- **Alignment 校正**：在训练完成后，通过一个小的校正数据集（均匀采样的无偏数据）对模型输出做 Platt Scaling 或 Temperature Scaling 校正。
- **实践经验**：多数团队在校正和不校正之间没有观察到巨大差异，但如果离线 AUC 与线上效果长期不匹配，偏差校正是一个值得排查的方向。

## 常见误区

- **负采样混入正样本**：这是最常见的"隐形炸弹"。由于数据流延迟、标签更新滞后等原因，采样时误将尚未标记的正样本当负样本。症状是离线指标正常但线上无效果。建议在数据 pipeline 中加正样本去重逻辑，并定期抽样检查。
- **全链路使用同一套负采样策略**：召回/粗排/精排的候选集大小和难度分布完全不同，负采样策略也应分别设计。把精排的负采样策略搬到召回上，会导致模型完全无法收敛。
- **曝光未点击 = 优质负样本**：在召回阶段恰恰相反，曝光未点击样本的分布已经严重偏移，用它做召回负样本会放大 Position Bias 和 Selection Bias。
- **负采样比例固定不变**：模型在训练不同阶段对负样本难度的需求不同。初期可多用简单负样本让模型快速收敛，后期逐渐增加困难负样本提升判别能力。
- **忽视采样后的样本权重校正**：非均匀负采样会在训练分布和真实分布之间引入偏差。严格做法是对负样本做 Importance Sampling 权重校正，或者至少在评估时意识到这种偏差的存在。
