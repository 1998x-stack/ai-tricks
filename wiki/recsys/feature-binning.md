# 特征分箱

## 是什么

特征分箱（Feature Binning）是将连续数值特征离散化为有限个区间（箱）的过程。例如，把用户的年龄从原始数值 18、25、34... 映射到 ["0-18", "19-25", "26-35", "36-50", "50+"] 这几个类别，再将类别做 Embedding 或 One-Hot 编码输入模型。

在推荐系统中，分箱最常见的方法是**等频分箱**：让每个箱中的样本数量大致相等。此外还有等距分箱、基于树模型的分箱、卡方分箱等方法。

### 分箱类型对比

| 分箱方法 | 优点 | 缺点 | 适用场景 |
|----------|------|------|----------|
| **等频分箱** | 每个箱样本均匀，模型训练稳定 | 边界可能不自然（等频不等距） | **推荐系统首选** |
| 等距分箱 | 边界直观，解释性强 | 可能存在空箱或样本极少的箱 | 数据分布均匀的场景 |
| 树模型分箱 | 自适应边界，信息量最大化 | 复杂度高，可能过拟合 | 特征工程后期精调 |
| 卡方分箱 | 基于统计检验，合并相似区间 | 实现复杂，超参多 | 风控等需要严格验证的场景 |

实践中，**绝大多数推荐系统场景用等频分箱足矣**。只在等频效果不佳时尝试其他方法。

## 为什么重要

**神经网络模型对于连续数值的处理是不如 XGBoost 的。** 这是一个常被忽略但极其重要的经验结论。原因在于：

- **非线性表达能力差异**：XGBoost 等树模型天然对连续特征做分段线性拟合（通过决策树的分裂），每个分裂点就是一个自适应学习的"分箱边界"。而神经网络的 Dense 层对连续值的处理本质上是线性变换 + 非线性激活，在拟合阶梯状的分段函数时效率远低于树模型。
- **数值尺度敏感**：神经网络对输入特征的尺度极其敏感。连续特征如果数值范围差异大（如"价格 0~10000"和"点击率 0~1"），不经过分箱直接输入会导致大数值特征主导梯度更新。
- **稀疏特征优先的推荐架构**：推荐系统以 Embedding + DNN 为主，本质上更适合处理离散化后的类别特征。将连续特征直接输入 DNN 反而违背了这一设计偏好。

分箱后的特征可以做 Embedding，相比原始数值输入，模型可以学到每个区间的独立表征，表达能力反而更强。

### 分箱与 Embedding 的组合优势

原始连续值在 DNN 中经过的是线性层，每个特征只有一个权重（一维标量），表达能力极为有限。分箱后再做 Embedding，相当于每个箱学到了一个多维向量（如 8 维或 16 维），模型的表达能力大幅增强。形式上：

```
原始连续值 -> Dense(1, hidden)           # 一个特征只有一个权重向量
分箱后 ID  -> Embedding(bucket_num, dim)  # 每个区间都有独立的向量表示
```

这也是为什么分箱后效果往往优于直接输入原始值——不是为了"离散化"本身，而是为了利用 Embedding 的丰富表达能力。

## 如何使用

### 等频分箱（默认首选）

将连续特征按照分位数划分为 K 个箱，每个箱包含等量的样本。

```python
import pandas as pd
# 等频分箱，每箱 10% 样本
df['price_bin'] = pd.qcut(df['price'], q=10, labels=False)
```

### 异常值单独成箱

观察数据分布后，将异常值（Outlier）单独分为一箱，而不是让它们与正常值混在一起。例如，某个特征的 99% 数据在 [0, 100] 区间，剩下 1% 在 [100, 100000] 区间，应该这样分箱：

```
箱1: [-inf, 0)    # 极少见的负值（可能是脏数据）
箱2: [0, 10]      # 常见范围
箱3: [10, 50]
箱4: [50, 100]
箱5: [100, +inf)  # 异常值范围
```

异常值单独分箱后，模型可以自己决定这个箱子是否重要（通过 Embedding 学习），而不会污染其他正常分箱的统计量。

### 等频分箱的完整流程

1. 特征预处理：处理缺失值（填充均值/中位数或单独成箱），截断极值（Winsorize）。
2. 计算分位数：对无缺失的数据做分位数计算，确定每箱边界。
3. 异常值单独成箱：将分布两端的极值分别合并为"低异常箱"和"高异常箱"。
4. IV/WOE 评估：对分箱结果计算 IV 值和 WOE（Weight of Evidence），判断每个箱的正负样本区分能力。
5. 合并或拆分：如果相邻箱体的 WOE 变化趋势一致，可以合并以减少稀疏性；如果某个箱的 IV 贡献过低，考虑拆分该箱范围的边界。
6. 上线前校验：确保分箱映射表能覆盖所有线上可能出现的新值，未被覆盖的 fallback 到最邻近的箱。

### IV 值评估分箱质量

IV（Information Value）可以定量评估分箱的好坏：

```
IV = Σ (P_i_good - P_i_bad) × ln(P_i_good / P_i_bad)
```

- IV < 0.02：特征对区分正负样本几乎无贡献
- IV 0.02 ~ 0.1：弱区分能力
- IV 0.1 ~ 0.3：中等区分能力
- IV > 0.3：强区分能力（但需警惕过拟合）

IV 值也帮助判断分箱是否合理——如果分箱后 IV 值偏低，说明分箱方式可能没有充分捕捉特征的信息量。可以尝试调整箱数或分箱边界（如改用等距分箱）。

### WOE 辅助判断分箱合理性

WOE（Weight of Evidence）衡量每个箱中正负样本占比的差异：

```
WOE_i = ln(P_i_good / P_i_bad)
```

- WOE 应随特征值的增大单调变化（递增或递减），如果出现非单调且无业务解释的跳变，说明分箱边界不合适。
- 相邻箱的 WOE 应呈稳定趋势，大幅波动说明分箱过细或该区间样本噪声过大。

### 箱数的选择

- 通常 10~20 箱是合理范围。
- 数据量越大，箱数可以越多。
- 箱数过少会损失信息，箱数过多则每个箱的样本稀疏，Embedding 难以学习。
- 也可以使用树模型（如 LightGBM）的叶节点作为分箱边界，做到自适应分箱。

### 不一定要分箱的场景

分箱并非适用于所有场景。以下情况可以直接使用原始连续值：

- **特征值本身就是离散且语义明确**：如"年龄"、"评分"等，装箱反而丢失原有的粒度信息。
- **模型结构已经设计了数值处理机制**：如使用了 Feature Transformer 或数值化专用网络（NODE、TabNet 等），这些结构对连续值的处理能力不亚于分箱。
- **特征维度极高且分箱后 Embedding 矩阵过大**：某些推荐模型的特征维度极高，分箱会额外增加 Embedding 的参数量，收益抵不上成本。

经验法则是：**先分箱试 10 箱，看 IV 值和离线 AUC 提升，如果无提升再回到原始值。** 不要假设分箱总是最优。

## 常见误区

- **直接把连续值塞进 DNN**：神经网络对连续值的处理效率不如树模型。多数推荐模型的标准做法是分箱 + Embedding，而不是直接输入原始数值。
- **不做异常值处理**：连续数据中的极端值会"压扁"其他分箱的范围。例如价格特征最大值 10 万，等频分箱后前 9 箱都挤在 [0, 100] 区间，最后一箱独占了 [100, 100000]。
- **分了箱就不需要归一化**：分箱后的特征是类别 ID，可以做 Embedding 或 One-Hot，不需要额外归一化。但如果同时保留原始连续值作为补充特征，仍需要对原始值做归一化。
- **分箱太多总比少好**：箱数过多会导致每个箱的样本量太少，Embedding 的统计意义下降，反而引入噪声。箱数应以"每个箱至少千级别样本"为底线。
- **一劳永逸，不重新评估**：数据分布会随时间漂移（如商品价格整体上涨），导致分箱边界不再合理。建议定期检查分箱边界的覆盖情况，必要时重新分箱。
- **忽视 IV/WOE 的单调性**：如果分箱后的 WOE 序列呈现非单调的波浪形，往往意味着分箱边界与正负样本分布不匹配，需要重新调整分箱方式。
