# 热启动

## 是什么

热启动（Warm Start）是指在已有训练好的"老汤模型"基础上，加载其全部或部分参数作为新模型的初始化，而非从头随机初始化进行训练。在推荐系统中，模型通常经过长期迭代积累了大量知识（Embedding 表征 + DNN 权重），热启动是模型迭代更新的标准做法。

与冷启动（从零训练）相对，热启动的核心假设是：旧模型已经学到了数据中的大部分有效模式，新模型只需在此基础上做增量调整。

### 热启动 vs. 增量训练 vs. 微调

这三个概念容易混淆，需要区分：

- **热启动（Warm Start）**：参数初始化手段。加载旧模型参数作为新模型的初始值，后续训练数据、目标、结构都可能改变。重点在于"初始状态"。
- **增量训练（Incremental Training）**：训练策略。用新数据继续训练已有模型，通常数据和目标不变，只是时间窗口推进。重点在于"持续更新"。
- **微调（Fine-tuning）**：在预训练模型基础上，用小数据做少量迭代适配特定任务。通常大幅降低学习率、减少训练步数。重点在于"小数据适配"。

推荐系统的热启动通常是上述三者的结合：加载老模型参数（热启动），用新一天的数据继续训练（增量训练），使用较小的学习率（微调）。

## 为什么重要

推荐系统的数据量通常极其庞大（数十亿到万亿级别），从头训练一次的计算成本极高。热启动可以：

- **大幅缩短收敛时间**：从零训练可能需要数个 epoch，而热启动通常在 1/10 到 1/5 的迭代步数内即可追平。
- **保留累积的领域知识**：老模型中的 Embedding 已经过长期训练，包含了用户行为、物料属性的丰富语义，丢弃这些信息是巨大的浪费。
- **稳定线上效果**：热启动后的模型在初始阶段的行为与旧模型接近，不会出现冷启动时的效果剧烈波动。

但需要特别注意的是：**热启动大概率只能追平旧模型，小概率找到之前没发现的关键特征可以超越**。如果期望热启动必然带来提升，那会失望。

### 热启动的本质：参数空间初始化而非增量学习

热启动的核心价值在于为优化过程提供一个好的起点。推荐系统的 Loss Landscape 是非凸的，存在大量局部最优。老模型已经落在了一个局部最优附近，热启动让新模型从该位置出发搜索相邻的更好的解。如果新模型引入的特征或结构变化很大，这个"好的起点"会失去意义，因为 Loss Landscape 本身已经变了。

从这个角度理解热启动的局限性就更直观了：它只保证你不倒退，不保证你前进。超越的关键在于新模型必须引入"老模型从未见过但有效的信息"。

## 如何使用

**核心原则：Embedding + DNN 必须全部加载，仅对倒数第一层做扰动。**

1. **全量加载**：同时加载老模型的 Embedding 层和 DNN 层的全部参数，不能只加载 Embedding 层。实验证明只热启 Embedding 层几乎必定负向，因为 Embedding 与 DNN 的高层语义已经耦合，只保留一半会破坏协同。
2. **最后一层扰动**：加载全部参数后，仅对倒数第一层（输出层）添加随机扰动或新增一个小型结构（类似 Wide&Deep 的 Wide 侧），让新模型在旧模型的基础上做微小调整。对多层做扰动几乎必定负向。
3. **与学习率配合**：热启动后通常使用比从头训练更小的学习率（建议缩小到原来的 1/5 到 1/10），避免大幅破坏已学好的参数。
4. **可选新结构**：如果需要在旧模型基础上引入新特征或新结构（如新增一个 feature cross 层），可效仿 Wide&Deep 的方式额外叠加一个小分支，仅在最后一层与旧模型汇合。

### 为什么最后一层扰动是关键

倒数第一层是模型输出层，它的参数与最终预测直接相关。扰动这一层等价于"保留旧模型的全部内部表征，只改变输出映射"。更深层的扰动会破坏中间层已经学好的特征交互模式，而这些模式是旧模型的核心知识。这种"只扰动一层"的策略在实践中稳定且低风险。

### 热启动后的验证流程

1. **对齐性验证**：热启动后先用一小批验证数据跑一轮，确认新模型的输出分布与旧模型接近。如果输出分布差异过大，说明加载配置有问题。
2. **对比实验**：同时维护一个从零训练的对照组，持续监控两组实验的离线指标差异。
3. **线上灰度**：热启动模型应逐步灰度（5% -> 20% -> 50% -> 100%），在每一步验证业务指标是否正向。热启动在灰度初期的表现尤其重要——如果前几天的指标就低于旧模型，说明热启动配置有问题。
4. **回滚预案**：热启动模型如果效果持续不达预期，保留旧模型的完整 checkpoint 和 Serving 管道，确保能快速回滚。

### 什么时候热启动可能超越

虽然热启动大概率只是追平，但以下场景存在超越的可能：

- **发现了之前被忽略的关键特征**：新模型加载了老模型的所有参数，额外加了 1~2 个信息量大的新特征，最后一层扰动给了新特征表达空间。
- **数据分布发生显著偏移**：老模型的参数提供了领域适应的"锚点"，新数据让模型在锚点附近微调到更适应新分布的位置。此时热启动的收敛速度和效果都优于冷启动。
- **修复了旧模型的系统性错误**：如果老模型存在已知的 Bias（如 Position Bias、Popularity Bias），热启动后在新数据上微调可以定向纠正这些偏差。

## 常见误区

- **只热启 Embedding 层**：看起来节省了计算量，但实践中效果几乎一定负向。Embedding 和 DNN 是联合训练的，单独加载 Embedding 相当于后端网络从零学，破坏了上下游的语义对齐。
- **扰动太多层**：对多层网络都做扰动，"新"的部分太多会导致热启动退化为冷启动，甚至因为初始状态不协调而比冷启动更差。
- **期望必能超越**：热启动的核心作用是"保底"——确保效果不低于旧模型，而非必然提升。超越需要新的有效特征或结构。
- **热启动与增量训练混用**：热启动是参数初始化手段，增量训练是训练策略。两者可以配合使用，但热启动后建议先用少量数据验证新模型是否对齐旧模型分布，再全量增量训练。
- **随意改变 Batch Size 或 LR 策略**：热启动时如果大幅改变 batch size 或学习率策略，会打乱已学好的 BN 统计量和梯度分布，导致热启动效果大打折扣。建议与原训练配置保持一致。
- **忽视输出层维度变化**：如果新模型需要预测更多目标（如从单任务变为多任务），输出层维度变了，热启动时需注意只加载匹配的层，新输出层用随机初始化。此时最后一层扰动必须重新设计。
- **不验证直接全量上线**：热启动后必须先验证对齐性再灰度，跳过验证直接全量上线是最危险的错误之一。
