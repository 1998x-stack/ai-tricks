# Focal Loss 详解

## 是什么

Focal Loss 是一种针对**类别极度不平衡**场景设计的损失函数，由 Lin 等人于 2017 年在 RetinaNet（目标检测论文）中提出。其核心思想是：降低易分类样本的损失贡献，让模型在训练中更关注那些难以分类的样本。

### 数学定义

标准交叉熵（Binary Classification）：
```
CE(p, y) = -log(p)    当 y=1
CE(p, y) = -log(1-p)  当 y=0
```

Focal Loss 在交叉熵基础上引入调制因子：
```
FL(p, y) = -α(1-p)^γ * log(p)    当 y=1
FL(p, y) = -(1-α)p^γ * log(1-p)  当 y=0
```

其中：
- **γ（Gamma）**：聚焦参数，控制易分类样本的权重衰减速度。γ=0 时退化为标准交叉熵。
- **α（Alpha）**：平衡权重，用于处理正负样本数量不均衡。α=0.25 是论文推荐的默认值。

调制因子 `(1-p)^γ` 的作用原理：当一个样本的预测概率 p 很高（即模型已经能轻松分类）时，`(1-p)^γ` 趋近于 0，该样本的 loss 贡献被大幅降低。反之，当 p 很低（模型难以分类）时，`(1-p)^γ` 接近 1，loss 几乎不受影响。

## 为什么重要

Focal Loss 的重要性体现在它解决了传统交叉熵在极端不平衡下的两个根本问题：

1. **易分类负样本主导梯度**：在目标检测等任务中，一张图片可能包含 100,000 个候选框，其中只有极少数包含目标物体。绝大多数候选框是被模型快速学会的"简单负样本"。在标准交叉熵下，这些大量易分类负样本的 loss 总和会淹没少数难分类正样本的信号。Focal Loss 通过降低易分类样本的权重，直接将训练焦点引导到少量难样本上。

2. **弥合了"分类"与"检测"的损失函数差距**：在 Focal Loss 提出之前，两阶段检测器（如 Faster R-CNN）通过 RPN 做候选框筛选来解决不平衡，而单阶段检测器（如 YOLO、SSD）直接全图预测，面临更严重的不平衡。Focal Loss 让单阶段检测器首次达到甚至超越了两阶段检测器的精度。

3. **广泛适用性**：虽然出身于目标检测，Focal Loss 已被广泛应用于文本分类中的长尾问题、医学图像中的罕见病检测、异常检测、以及 NLP 中的关系抽取等场景。

## 如何使用

### Gamma 参数调优

Gamma 是 Focal Loss 最重要的超参数。实践经验表明：

- **γ=0**：退化为标准交叉熵，对不平衡无特殊处理。
- **γ=0.5**：轻微聚焦，适用于轻度不平衡（正负比 1:10 ~ 1:50）。
- **γ=1.0**：中等聚焦，常见起始值。
- **γ=2.0**：强聚焦，RetinaNet 论文默认值，适用于中高度不平衡（正负比 1:100 ~ 1:1000）。
- **γ=5.0**：极强聚焦，适用于极端不平衡（正负比 1:10000+）。

**推荐搜索策略**：以 10 倍步长搜索。即先试 γ=0.5，效果好则降为 γ=0.2、0.1；效果差则升为 γ=2、5。这个"×10 衰减"的经验法则来自实践中发现的 Gamma 敏感度模式——Gamma 对结果的影响大致呈对数尺度。

### Alpha 参数

Alpha 用于平衡正负样本数量，在 Gamma 的基础上额外提供一重调节。常用设置：
- 当正样本极为稀少时，设 α > 0.5 来提高正样本权重。
- 论文默认 α=0.25（即正样本权重 0.25，负样本权重 0.75）——注意这不意味着正样本不重要，而是因为 Gamma 已经在压低易分类负样本的权重了，α 主要用于精细调节。
- 实践中可以固定 γ 后网格搜索 α：[0.1, 0.25, 0.5, 0.75]。

### 代码实现（PyTorch）

```python
import torch.nn.functional as F

def focal_loss(logits, targets, gamma=2.0, alpha=0.25):
    ce_loss = F.binary_cross_entropy_with_logits(logits, targets, reduction='none')
    p = torch.sigmoid(logits)
    pt = p * targets + (1 - p) * (1 - targets)  # pt = p if y=1 else 1-p
    focal_weight = (1 - pt) ** gamma
    if alpha >= 0:
        alpha_t = alpha * targets + (1 - alpha) * (1 - targets)
        focal_weight = alpha_t * focal_weight
    return (focal_weight * ce_loss).mean()
```

## 与类别加权的对比

| 维度 | Focal Loss | 类别加权（Class Weighting） |
|------|-----------|---------------------------|
| 机制 | 动态降低已学好样本的权重 | 静态增大少数类样本的权重 |
| 适应性 | 自适应——同一个样本在训练中后期权重会自动降低 | 固定——从头到尾权重不变 |
| 调节参数 | Gamma（聚焦程度）+ Alpha（平衡） | 仅权重比例 |
| 适用场景 | 极端不平衡（1:1000+）、大量易分类负样本 | 轻度/中度不平衡，简单场景 |
| 计算开销 | 几乎为零（仅多一次幂运算） | 几乎为零 |
| 与模型结构耦合 | 无——可插入任何分类模型 | 无 |

核心区别：类别加权对所有少数类样本一视同仁地提高权重——不管它是否已经被模型学会。Focal Loss 则会动态降低已学会样本的权重，这让它在极端不平衡下往往比类别加权更有效。但在轻度不平衡下（正负比 < 1:10），类别加权更简单、稳定，是首选选择。

**组合使用**：实践中可以将两者结合——在类别加权的基础上叠加 Focal Loss，尤其适合 α 固定为 0.5 而主调节由 Gamma 完成的情况。

## 常见误区

### 误区一：Focal Loss 能替代一切不平衡处理手段

Focal Loss 是损失函数层面的解决方案，但不替代数据层面的处理。当数据极度不平衡时，仍需配合[[./class-imbalance|过采样/欠采样]]等数据增广手段。

### 误区二：Gamma 越大越好

Gamma 过大会导致模型过度关注噪声样本（那些"难"是因为标注错误而非本质难分的样本），损害泛化能力。始终从 γ=0.5 或 1.0 开始搜索，不要默认使用 2.0。调试时应[[../debugging/debugging|对比使用 Focal Loss 前后难样本的梯度分布]]，确认调制因子确实改变了训练焦点而非引入噪声。

### 误区三：Focal Loss 适用于所有任务

Focal Loss 主要适用于二分类和目标检测。多分类场景中需要调整实现（将 γ 应用于每个类别的 softmax 概率）。在回归任务、排序任务（如 NDCG 优化）中不适用。

### 误区四：只在正样本少的场景用

即使正负样本数量均衡，如果存在大量"简单样本"，Focal Loss 也可能有所帮助。反之，如果数据已经是 hard mining 采样后的结果（如难负样本挖掘后的数据集），Focal Loss 的效果会大打折扣。
