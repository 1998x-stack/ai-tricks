# 类别不平衡解决方案

## 是什么

类别不平衡（Class Imbalance）是指分类任务中各个类别的样本数量相差悬殊的现象。常见比例从 1:10（轻度不平衡）、1:100（中度不平衡）到 1:10,000+（极端不平衡）。不平衡问题广泛存在于：

- **异常检测**：正常交易占 99.9%，欺诈交易占 0.1%
- **医学诊断**：健康人群远多于罕见病患者
- **目标检测**：一张图中背景候选框数量远超物体候选框
- **长尾识别**：少数"头部"类别占据了大部分数据，"尾部"类别样本极少
- **文本分类**：某些类别（如投诉、举报）天然稀少
- **点击率预测**：用户未点击的样本远多于点击样本

不平衡的程度决定了应对策略的激进程度：1:10 轻度不平衡可能仅需类别加权，而 1:10,000+ 的极端不平衡可能需要 Focal Loss + 过采样 + 异常检测转换组合使用。

## 为什么重要

类别不平衡会导致一个隐蔽但严重的问题：**模型学会"偷懒"**——将所有样本预测为多数类即可获得很高的准确率，但这个模型完全无用。

例如：一个 99:1 的正负比数据集，无脑全预测为正类就能达到 99% 的准确率，但这样的模型在真实场景中无法识别任何一个负类。这正是为什么[[./eval-metrics|不能只看准确率]]——在不平衡数据上，准确率是一个极其容易欺骗人的指标。更隐蔽的情况是：模型准确率看起来不错（如 95%），但少数类的召回率可能只有 5%，这种模型在欺诈检测等场景中完全不可用。

不平衡问题不解决，模型优化的方向就从根本上偏了。标准交叉熵损失被大量多数类样本主导，少数类的梯度信号被淹没，导致模型对少数类的决策边界片面甚至完全坍塌。[[../debugging/debugging|调试的起点]]往往是先确认数据分布——如果训练过程中 loss 在下降但少数类的 F1 完全没有改善，大概率是类别不平衡的问题没有被正确处理。

## 如何使用

类别不平衡的解决方案可分为三个层次，推荐按以下优先级尝试：

### 第一优先：损失函数层面

**Focal Loss**：在极端不平衡（1:1000+）场景下，推荐直接使用 [[./focal-loss|Focal Loss]]。它通过调制因子 `(1-p)^γ` 动态降低易分类样本的权重，让模型聚焦于难样本。Gamma 参数从 0.5 开始搜索，以 10 倍步长递增（0.5 -> 2 -> 5）。Focal Loss 尤其适合正样本少但"难"正样本多的场景，如目标检测中密集的候选框分类。

**类别加权（Class Weighting）**：在轻度到中度不平衡（1:10 ~ 1:100）场景下，类别加权是更简单、更稳定的选择。使用 sklearn 自动计算权重：

```python
from sklearn.utils.class_weight import compute_class_weight
weights = compute_class_weight('balanced', classes=np.unique(y), y=y)
# 返回的权重 = n_samples / (n_classes * np.bincount(y))
```

**组合使用**：Focal Loss 的 Alpha 参数 + 独立的类别权重可以叠加使用，在极端不平衡下比单独使用任一种更有效。具体做法：将类别权重作为 loss 的乘数，再应用 Focal Loss 的调制因子。

**Cost-Sensitive Learning**：不仅在损失函数层面加权，在模型训练的其他环节（如决策阈值、采样策略）也体现少数类的更高代价。本质上是对"误分类少数类的代价远高于误分类多数类"这个业务约束的数学表达。

### 第二优先：数据层面

**过采样（Oversampling）**：对少数类重复采样，使其数量与多数类接近。最简单的实现是 `from imblearn.over_sampling import RandomOverSampler`。注意：纯随机过采样容易导致对少数类过拟合（模型反复看到相同的少数类样本）。过采样倍数一般不要超过 10 倍，否则重复样本过多会损害泛化能力。

**SMOTE（Synthetic Minority Oversampling）及其变体**：在少数类样本之间插值生成合成样本，避免简单复制。`from imblearn.over_sampling import SMOTE`。SMOTE 对低维连续特征效果较好，但在高维稀疏数据（如文本 BoW 特征）中可能生成无意义的样本。变体包括 Borderline-SMOTE（在决策边界附近采样）、ADASYN（自适应生成不同数量的合成样本）。

**欠采样（Undersampling）**：随机删除多数类样本以使类别平衡。适用于数据量极大的场景（100万+），但会丢失多数类中可能有用的信息。改进方案包括 NearMiss（选择多数类中距离少数类最近的样本）和 Tomek Links（删除不同类别之间的边界样本以提纯决策边界）。

**针对性数据增强**：对少数类做更强的[[./data-augmentation|数据增强]]，生成更丰富的变体；对多数类做更弱的增强或不做增强。这在视觉任务中特别有效——少数类的旋转、裁剪变体可以产生新的有效训练样本，比重复复制更有信息量。

**硬负样本挖掘（Hard Negative Mining）**：在训练过程中迭代地找出那些被模型错误分类的多数类样本（即"困难负样本"），将它们加入训练集中。这在[[../data-eval/data-eval|召回和对比学习任务]]中有显著效果。实现方式：先用少量数据训练一个初始模型，在全量数据上推理找出 false positive 样本，将这些样本加入训练集重训。

### 第三优先：算法与评估层面

**集成方法**：将多数类分成多个子集，每个子集与全部少数类组成平衡数据集训练一个子模型，最后集成所有子模型的预测（EasyEnsemble、BalanceCascade）。这种方法保留了多数类的全部信息，同时每折的训练分布接近平衡。

**解耦训练（Decoupled Training）**：先使用原始（不平衡）数据训练特征提取器，再冻结 backbone，仅用平衡数据重训分类头。这种方法适用于长尾识别任务，因为特征表示可以在全量数据上学习，而分类头在平衡数据上校准。

**阈值移动（Threshold Moving）**：训练时不处理不平衡，只在推理时调整决策阈值（从默认的 0.5 向下调低，使得更多样本被分为正类）。最优阈值可以通过验证集上的 Precision-Recall 曲线找到。

**异常检测视角转换**：当不平衡极其严重（1:10000+）时，放弃分类框架，转为异常检测（One-Class SVM、Isolation Forest、AutoEncoder）——将少数类视为异常点而非分类目标。这种方法在欺诈检测和网络入侵检测中常见。

**评估指标切换**：停止使用准确率。必须使用 [[./eval-metrics|F1、AUC、Precision-Recall Curve]] 等对不平衡鲁棒的指标进行模型选择和调参。

## 常见误区

### 误区一：过采样后直接用准确率评估

过采样改变了验证集的分布。如果在过采样后的验证集上评估，指标会虚高。评估必须在保持原始分布的真实测试集上进行。正确的流程是：只在训练数据上做不平衡处理，测试集保持原始分布不变。

### 误区二：欠采样丢失太多信息

欠采样确实会丢弃多数类样本，但丢弃的不一定是有用信息——如果多数类中有大量冗余样本（如 99% 的正常交易都是高度相似的），欠采样可能不影响甚至提升模型性能。关键在于：欠采样后再配合集成（如 BalanceCascade）可以弥补信息损失。不要轻易放弃欠采样，关键是找到合适的保留比例。

### 误区三：SMOTE 是万能药

SMOTE 在以下场景中效果有限：高维数据（如文本、图像特征空间）中，合成样本可能落在高维空间的无关区域；数据极度稀疏时，插值两点几乎都在零向量附近，生成样本无意义；标签本身有噪声时，SMOTE 会放大噪声影响——若合成样本使用了错误的标签，会引入更大的噪声。

### 误区四：只在训练时处理不平衡，评估却用准确率

这是一个系统性错误。没有正确的评估指标，无论你用什么方法处理不平衡，调参的方向都是错误的。准确率会在不平衡数据上给出乐观到荒谬的结论。请参考[[./eval-metrics|评估指标指南]]选择适合不平衡场景的指标。

### 误区五：不平衡问题只存在于二分类

多分类中的长尾问题同样严重。头部类别（几百到几千样本）主导训练过程，尾部类别（个位数样本）几乎学不到任何有用的决策边界。解决方法类似：按类别频率分层加权、对尾部类别做更强的增强、或者使用解耦训练策略（Decoupled Training）。
