# 数据增强完全指南

## 是什么

数据增强（Data Augmentation）是在不收集新数据的前提下，通过对现有训练数据施加**保持标签语义不变**的变换来生成更多训练样本的技术。它的本质是将人类先验知识（如"图像翻转后仍是同一物体"、"删除无关词后语义不变"）编码到数据分布中。

### 常见分类

按数据模态分为两大阵营：

**CV 领域**：
- **几何变换**：随机裁剪、水平/垂直翻转、旋转（±10°~±30°）、缩放、平移、切变。
- **色彩/光照变换**：亮度调整、对比度调整、饱和度调整、色相抖动、高斯噪声。
- **高级混合**：MixUp（两张图按比例混合像素和标签）、CutMix（一张图切一块贴到另一张图上）、CutOut/Random Erasing（随机遮挡区域）。
- **自动增强**：AutoAugment、RandAugment——学习最优增强策略或随机组合多种增强。

**NLP 领域**：
- **词级变换**：同义词替换、随机插入/删除/交换词语。
- **回译（Back-Translation）**：将句子翻译成中间语言再翻译回来，生成语义相似但表达不同的句子。
- **句子级变换**：随机截断、拼接、TF-IDF 引导的词语替换。
- **LLM 生成**：使用大模型生成语义相似但表述不同的变体数据。

## 为什么重要

数据增强是深度学习泛化能力的核心支柱：

1. **数据量乘以系数**：一张原始图片通过随机裁剪+翻转+色彩抖动可以产生数十个有效变体，相当于免费将数据集扩大数倍到数十倍。在数据标注成本高昂的现实场景中，这是回报率最高的"投资"。

2. **内置正则化**：数据增强为模型提供了隐式的正则化效果——模型无法记住固定的像素模式，必须学到与增强无关的语义特征。[[../regularization/regularization|这与 Dropout、Weight Decay 等正则化手段互为补充]]。

3. **提升鲁棒性**：经过增强训练后的模型对光照变化、遮挡、视角变化等实际部署中的常见扰动有更强的抵抗力。这是模型从"在测试集上表现好"走向"在真实环境中表现好"的关键一步。

4. **缓解类别不平衡**：对于[[./class-imbalance|样本不足的少数类]]，可以通过增强来生成更多变体，比单纯的重复抽样更有信息量。

## 如何使用

### CV 增强使用指南

**基础配置（适用于大多数视觉任务）**：
```python
from torchvision import transforms

train_transform = transforms.Compose([
    transforms.RandomResizedCrop(224, scale=(0.8, 1.0)),
    transforms.RandomHorizontalFlip(p=0.5),
    transforms.ColorJitter(brightness=0.2, contrast=0.2, saturation=0.2, hue=0.1),
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])
])
```

**增强强度本身就是超参数**。对每个数据集而言，最优增强强度是不同的——过强会破坏原始信号（如裁剪过多导致关键物体被切掉），过弱则正则化效果不足。建议将增强参数纳入超参数搜索范围，例如裁剪 `scale` 的下界从 0.5 到 0.9 搜索。

**检测/分割任务注意事项**：几何变换（翻转、旋转、裁剪）需要同步变换边界框或掩码。像素级变换（色彩抖动、噪声）则不需要。MixUp 和 CutMix 常用于分类任务，在检测中需要更谨慎地处理标签。

### NLP 增强使用指南

**回译推荐配置**：
```python
# 使用 Hugging Face 的翻译 pipeline
from transformers import pipeline
back_translator = pipeline("translation", model="Helsinki-NLP/opus-mt-zh-en")
en_text = back_translator("今天天气很好")[0]['translation_text']
back_to_zh = pipeline("translation", model="Helsinki-NLP/opus-mt-en-zh")
original = back_to_zh(en_text)[0]['translation_text']
```

回译通常能产生质量最高的增强数据，但计算成本较高。轻量替代方案包括同义词替换（基于 WordNet 或同义词词典）和随机删除（随机删除 10%-20% 的词语）。

### 何时使用

- **数据量不足**：训练集 < 10K 时，增强的必要性极高。
- **过拟合信号明确**：训练 loss 持续下降而验证 loss 不再下降时，增加或增强增强强度往往是最有效的第一步。
- **数据分布单一**：训练数据在某个维度（如光照、角度）上缺乏多样性时，针对性增强该维度。

## 常见误区

### 误区一：增强越多越好

过度增强会引入噪声甚至改变语义。例如：旋转 90° 以上会使"直立的人"变成"躺着的人"——标签从"站立"变为非预期状态；对 OCR 任务，垂直翻转完全改变了文字的可读性；对医学影像，过强的色彩抖动可能抹去病灶特征。增强的边界是**标签语义不变**。

### 误区二：训练和测试使用不同的预处理

这是一个实现层面常见但后果严重的 bug。测试或推理时必须使用与训练一致的预处理（相同的 resize 尺寸、相同的归一化参数），但**不能在测试时使用随机的数据增强**（如随机裁剪、随机翻转）。测试时使用固定的中心裁剪。

### 误区三：NLP 增强会改变原本正确的语义

同义词替换可能改变语境含义（"苹果"在"吃苹果"和"苹果公司"中含义不同，直接替换可能导致语义偏差）。回译可能丢失领域术语（专业名词在翻译后再翻译回来会变成通用表达）。NLP 增强后必须做人工抽检，确认语义未偏离。

### 误区四：忽略增强与模型架构的兼容性

ViT 与 CNN 对增强的敏感度不同——ViT 通常需要更强或更多的正则化和增强（如 RandAugment、MixUp、CutMix），否则容易过拟合。数据增强策略需与模型结构匹配。
