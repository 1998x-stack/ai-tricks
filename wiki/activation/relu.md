# ReLU 激活函数

## 是什么

ReLU (Rectified Linear Unit，修正线性单元) 是深度神经网络中最常用的
激活函数，定义为：

$$f(x) = \max(0, x)$$

即输入为正时原样输出，输入为负时截断为 0。导数在正半轴恒为 1，
负半轴恒为 0。在 0 点处不可微，但实践中取左导数或右导数均可，
不影响训练收敛。

ReLU 看似简单，却是深度学习在 2010 年代取得突破的关键因素之一。
在 ReLU 之前，sigmoid 和 tanh 是主流激活函数，深层网络训练极其困难——
2011 年之前几乎无法可靠地训练超过 5 层的全连接网络。
2011 年 Glorot 等人发表的《Deep Sparse Rectifier Neural Networks》
首次系统论证了 ReLU 可以显著改善深层网络的训练，此后 ReLU 迅速
成为 CV 领域隐层的默认激活函数。AlexNet 在 2012 年使用 ReLU 取得
ImageNet 冠军，标志着深度学习时代的真正开端。

ReLU 的设计哲学也启发了后续一系列改进变体，参见 [[leaky-relu-prelu]]。

## 为什么重要

**缓解梯度消失**：sigmoid/tanh 在饱和区梯度趋近于 0，深层网络反向传播时
梯度连乘后迅速消失。例如一个 10 层的 sigmoid 网络，每层梯度最大 0.25，
连乘后梯度接近 $0.25^{10} \approx 9.5 \times 10^{-7}$。ReLU 在正半轴
梯度恒为 1，梯度可以无损传播到浅层，这是它能训练深层网络的根本原因。
深度从 5 层扩展到 20+ 层成为可能，直接推动了 AlexNet、VGG（19 层）、
ResNet（152 层）等里程碑架构的出现。

**计算效率极高**：ReLU 仅涉及一次比较和一次取最大值操作，没有指数运算。
相比 sigmoid（包含 exp）和 tanh（包含 exp 和除法），ReLU 的前向和反向
计算都几乎零开销。在大型卷积网络中，激活函数的计算时间可从占总时间的
30% 降至接近 0，显著加速训练和推理。这对大规模实验和大规模部署来说
意义重大。

**稀疏激活**：ReLU 将负输入截断为 0，使神经元以约 50% 的概率失活，
产生稀疏表示。稀疏性有助于解耦特征、减少过拟合，且稀疏矩阵在工程上
更易优化。相比之下，sigmoid/tanh 的输出几乎总是非零，冗余激活增加了
计算负担。实验表明，ReLU 网络的稀疏性通常在 50-80% 之间，这意味着
大部分计算资源集中在真正有响应的特征上。

**收敛速度**：ReLU 的非饱和特性使得梯度不会衰减，实验表明其收敛速度
显著快于 sigmoid/tanh。这使得我们可以训练更深的网络而不依赖逐层预训练，
大幅简化了训练流程。

## 如何使用

**隐层默认首选**：所有深度神经网络的中间隐层，尤其 CV 领域，直接
使用 ReLU 作为默认选择。搭配 [[../initialization/initialization]] 中的 He 初始化
（方差 $2/n$）效果最佳。ReLU + He 初始化的组合已被证明是训练深度网络
最稳定的基线方案，比 ReLU + Xavier 初始化的收敛速度快数倍。

**与 BN 配合**：ReLU + Batch Normalization 是标准组合。BN 将输入
拉回到 0 附近，确保大部分神经元处于激活状态，减少 dying ReLU 风险。
同时 BN 降低了对初始化精度的依赖，两者搭配使训练更鲁棒。现代 CNN
几乎都使用 Conv → BN → ReLU 的顺序。参见 [[../normalization/normalization]]。

**学习率控制**：ReLU 的学习率不能设太大，否则大量神经元会陷入永不
激活的状态。建议在可工作的最大学习率下调小 20-30%，平衡收敛速度与
神经元存活率。如果发现训练过程中激活率持续下降，应降低学习率。
Warmup 策略（先从小学习率逐步升高）也是保护 ReLU 神经元的有效手段。
参见 [[../optimizer-lr/optimizer-lr]]。

**Conv + MaxPool + ReLU 顺序**：由于 ReLU 和 MaxPool 都是 max 操作，
Conv → MaxPool → ReLU 在数学上等价于 Conv → ReLU → MaxPool，
但前者少做一次逐元素操作，节省计算量。在 CNN 中推荐
Conv → MaxPool → ReLU。

**输出层禁用**：最后一层切勿使用 ReLU。ReLU 会将负值截断为 0，
连续值预测会被破坏。输出层应根据任务类型选择 softmax / sigmoid / 线性，
详见 [[activation-selection]]。

**新模型调试**：做新模型时，最开始不要加激活函数。先构建纯线性模型
确认数据流和梯度正常，再逐步引入 ReLU。每一步只引入一个变化，
防止问题无法定位。

**欠拟合时**：如果模型欠拟合，除了增加网络宽度和深度外，也可考虑
在特定层加入 ReLU/Mish 等激活函数来增加非线性表达能力。不过先确认
欠拟合不是因为训练不足或学习率问题。参见 [[../capacity/]]。

## 常见误区

**ReLU 不会梯度消失所以万事大吉**：ReLU 解决了正半轴的梯度消失，
但负半轴梯度恒为 0，会导致神经元失活（dying ReLU）。当学习率过大
或初始化不当时，大量神经元可能永久死亡，网络容量骤降。这不是梯度消失，
而是"梯度死亡"——两者的根本区别在于前者是梯度趋近于 0，后者是梯度
精确为 0。梯度死亡无法通过增加训练轮数解决。

**ReLU 不适合所有任务**：ReLU 输出无上界，不适合需要概率输出的场景。
在 RNN 中 ReLU 容易导致输出爆炸——循环权重连乘使正激活值指数增长。
因此 RNN 默认用 tanh 而非 ReLU。且 ReLU 在负半轴的硬截断丢失了
负信息，某些任务（如音频信号、自编码器）中负信息是重要的。

**ReLU = 无脑选，不需要调参**：ReLU 虽然好用，但初始化和学习率仍需
配合。错误搭配（如 ReLU + Xavier 初始化）会导致梯度爆炸——因为
Xavier 的方差设计假设激活函数是对称的（输出均值约 0），而 ReLU 只有
正半轴输出，方差减半但 Xavier 并未补偿，导致深层方差累积。
学习率过大则神经元大面积死亡。

**ReLU 输出无限大也没关系**：ReLU 无上界，当网络层数极深或权重更新
剧烈时，激活值可能不断增大。在极端情况下这会导致数值溢出或训练不稳定，
此时可考虑 ReLU6 限制上界。对于需要量化的移动端部署，无上界也是问题——
量化需要确定动态范围。
