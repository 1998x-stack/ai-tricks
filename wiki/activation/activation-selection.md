# 激活函数选型指南

## 是什么

激活函数的选型需要考虑网络位置（隐层 vs 输出层）和任务类型（分类/回归/二分类/排序等）。错误的选型可能导致梯度消失、信息丢失或训练不收敛。本文提供从网络结构到业务场景的完整选择框架。

## 为什么重要

激活函数决定了神经网络的表达能力、训练效率和最终性能。选型不当的典型后果：

- **隐层选错**：sigmoid/tanh 在深层网络中梯度消失，网络深度超过 5 层基本无法训练；ReLU 在 RNN 中可能导致输出爆炸
- **输出层选错**：多分类用 sigmoid 输出概率和不等于 1，回归用 softmax 强制类别分布，loss 永远降不下去
- **配对错误**：激活函数与 [[../initialization/initialization]] 必须匹配——ReLU 配 He，tanh/sigmoid 配 Xavier，否则梯度可能在训练一开始就爆炸或消失
- **盲目尝试**：业务场景中过早更换激活函数是常见的浪费时间方式，应先做好数据、学习率和训练流程。参见 [[./activation]] 中"调参顺序"和"业务场景不宜过早更换激活函数"的讨论

## 如何使用

### 输出层选型（按任务类型）

| 任务类型 | 激活函数 | 原因 |
|---------|---------|------|
| 多分类 (N 类) | Softmax | 输出和为 1 的概率分布，argmax 得类别 |
| 二分类 | Sigmoid | 输出 (0,1) 可视为概率，配合 BCELoss |
| 多标签分类 | Sigmoid（每个输出单元独立） | 每个标签独立概率，互不排斥 |
| 回归（连续值） | 线性 / identity / 无激活 | 输出无界，允许任意实数 |
| 回归（非负） | ReLU / Softplus | 确保输出 ≥ 0（如价格、年龄、长度） |
| 回归（(0,1) 区间） | Sigmoid | 输出仅需归一化到 (0,1) |
| 召回排序 | Softmax（listwise） | 推荐召回场景，softmax 的对比学习效果优于 sigmoid 的点式二分类 |

**最后一层永远不要用 ReLU**：ReLU 截断负值为 0，连续值预测直接丢失信息，即使预测正值的任务也会损失梯度信号。参见 [[relu]]。

### 隐层选型

| 场景 | 推荐激活 | 原因 |
|------|---------|------|
| CNN / MLP 通用隐层 | ReLU | 收敛快、缓解梯度消失、默认首选 |
| RNN / 序列模型 | tanh | 控制输出范围 (-1,1)，防止 RNN 梯度爆炸 |
| Transformer / BERT | GELU | 平滑的 ReLU 近似，训练更稳定 |
| 自编码器（隐层） | ReLU | 稀疏编码，解耦特征 |
| 自编码器（输出） | 线性 / Sigmoid（取决于输入类型） | 重构任务，匹配输入分布 |
| 图神经网络 | ReLU / LeakyReLU | 节点特征变换 |
| 深度网络（>100 层） | ReLU + BN / 残差连接 | 极度依赖梯度流通 |
| 移动端/轻量网络 | ReLU6 | 限制激活值上界，降低量化难度（MobileNet 系列常用） |

### 调试新模型的起点策略

做新模型时，**最先不要加激活函数**：先构建纯线性模型确认 baseline，确认数据流、梯度形状、loss 下降趋势正确后，再逐步加入 ReLU → 观察收敛变化 → 如有 dying ReLU 现象就切换到 [[leaky-relu-prelu]]。每一轮只引入一个变化，防止多个变量同时变动导致问题无法定位。如果加了激活函数后 loss 反而上升，检查初始化是否正确。

### 调参优先级

正确的调参顺序：数据质量 → 学习率 → 训练流程（epoch、batch size）→ 网络结构（层数、宽度）→ 激活函数 → 正则化。激活函数的调优优先级不高，先用 ReLU 做 baseline，精度稳定后再尝试变体来微调。许多人一看效果不行就改激活函数，这通常不是最高 ROI 的投入。

### 损失函数的配对

输出层激活函数需要与 [[../loss/]] 配对：

| 输出激活 | 对应损失 |
|---------|---------|
| Softmax | CrossEntropyLoss / NLLLoss |
| Sigmoid（二分类） | BCELoss / BCEWithLogitsLoss |
| Sigmoid（多标签） | BCEWithLogitsLoss（每个标签独立） |
| 线性（回归） | MSELoss / L1Loss |

Softmax + CrossEntropyLoss 内部已经包含了 log_softmax 计算，不要在 loss 前手动再做 softmax。

## 常见误区

**Softmax 用在回归任务**：回归任务是输出连续值，不是类别概率。用 softmax 会强制输出和为 1，对所有样本惩罚类别分布差异，完全错误的 loss landscape，训练永远不会收敛。

**ReLU 用在输出层做回归**：即使预测的是正数（如图像像素值），ReLU 也会在反向传播时丢失负半轴梯度信息。更重要的是 ReLU 输出可以非常大（无上界），不适合做概率或标准化输出。区分"非负输出"和"输出层无限制"两个需求。

**总是用 sigmoid + 二分类交叉熵**：对二分类，Sigmoid + BCELoss 是正确的。但对多分类，用 Sigmoid + BCELoss（而不是 Softmax + CrossEntropyLoss）意味着多标签分类，两者不等价。多分类始终选择 Softmax。

**RNN 用 ReLU**：理论上 ReLU 在 RNN 中也有效，但实践中 RNN 的循环权重连乘容易使 ReLU 输出爆炸。PyTorch 的 RNN/LSTM 默认使用 tanh，这是一个经过充分验证的选择，不要轻易替换。如果确实需要在 RNN 中用 ReLU，必须配合梯度裁剪。

**隐层和输出层用相同的激活函数**：隐层用 ReLU 效果好，不代表输出层也应该用 ReLU。相反，输出层永远不应该用 ReLU。输出层的选择完全由任务类型决定，与隐层的选择无关。
