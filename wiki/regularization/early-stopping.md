# Early Stopping

## 是什么

Early Stopping（早停）是在训练过程中监控验证集指标（通常是 loss），当指标连续若干轮不再改善时提前终止训练的方法。本质上它通过限制训练轮次来防止过拟合——模型在验证 loss 开始回升之前停止，也就不会在训练集上继续死记硬背。

早停的核心参数是 **patience（耐心值）**：即验证 loss 连续多少轮不下降或不再改善时才停止。此外还有 `min_delta`（最小改善阈值，忽略微小震荡）和 `restore_best_weights`（是否恢复到验证集最优的 checkpoint）。

### 早停的触发条件（伪代码）

```
if val_loss > best_val_loss - min_delta:
    counter += 1
    if counter >= patience:
        stop training
else:
    counter = 0
    update best_val_loss
```

## 为什么重要

- **防止过拟合的最后一道防线**：即使其他正则化手段设置不当，早停也能在过拟合严重之前截停训练
- **节省计算资源**：不需要预设固定的总轮次，模型收敛后自动停止，避免无效计算浪费
- **超参数搜索中的筛子**：在大型超参数搜索或架构搜索中，早停能快速淘汰表现差的配置，节省大量 GPU 时数
- **自适应训练时长**：不同配置的收敛速度差异很大，早停让每个实验自动找到自己的最优训练轮次

## 如何使用

### Patience 选择

patience 的选择取决于任务的验证指标曲线的平滑度：

| 场景 | 推荐 patience | 说明 |
|------|--------------|------|
| 图像分类（CV） | 10~20 | 验证 accuracy/loss 曲线相对平滑 |
| NLP / Transformer 微调 | 3~10 | 收敛较快，耐心值不宜过高 |
| 强化学习 | 20~50+ | 曲线震荡大，需要更大 patience |
| 推荐系统 | 1~5 | 可能一个 epoch 就过拟合 |
| 大型超参数搜索 | 3~5 | 快速淘汰明显垃圾组合 |

**经验方法**：先跑一轮完整训练观察验证 loss 曲线的波动幅度，用波幅的 3~5 倍作为 patience。如果曲线在 5 个 epoch 内波动 ±2%，那么 patience 设为 10~15 比较合理。

### ASHA / Hyperband 与早停

[[regularization]] 中提到 ASHA 和 Hyperband 是早停在超参数搜索中的进阶应用：

- **ASHA（Asynchronous Successive Halving Algorithm）**：异步版的 Successive Halving，不断分配预算给有潜力的配置，提前停止表现差的
- **Hyperband**：在不同预算水平上运行 ASHA，解决"分配多少预算给每个配置"的元问题

这两种方法的核心假设是：**如果某组超参数在少量 epoch 后效果已经很差，那么给它更多 epoch 也不会变好**。早停短训适合筛掉明显垃圾组合，但不能完全替代完整训练——有些配置确实需要更长的训练才能展现优势。

### 随机种子策略

早停可以用来做种子选择：

1. 对一批随机种子，每颗种子都跑早停训练（少量 epoch）
2. 挑出 loss 下降速度最快的几颗种子
3. 只对筛选出的种子做深度训练

这在训练不稳定、不同种子方差大的任务中非常有效。

### 使用 restore_best_weights

始终设置 `restore_best_weights=True`（在 PyTorch Lightning、Keras 中对应参数）。早停发生在验证 loss 已经回升之后，如果不在停止时恢复到最佳权重，你保存的模型实际上已经过拟合了。

## 不要过早停止

这是早停中最重要也最容易被忽略的反向观点。[[../../contradictory]] 详细讨论了何时不应该早停。关键场景：

### 收敛可能发生在训练后段

某些训练配置的 loss 曲线存在"高原期"——前几十个 epoch 看似 plateau，随后突然下降。这种情况在：
- 使用 SGD（非 Adam）训练时常见
- Warmup + Decay 学习率调度中，warmup 阶段之后 loss 才开始下降
- Transformer 和大型模型训练的前期热身阶段
- 多阶段训练的前期预训练阶段

### 先过拟合再早停

不要跳过"过拟合验证"直接上早停。正确流程：
1. 先确保模型能过拟合训练集（确认代码正确 + 模型容量足够）
2. 加上早停作为防止过度训练的保护
3. 如果早停过早触发，增加 patience 或检查学习率是否过高

早停是**锦上添花的保护措施**，不是最终手段——如果你的模型根本学不会（欠拟合），早停只会让你停在更差的位置。

### ASHA/Hyperband 的误杀风险

ASHA/Hyperband 虽然高效，但存在过早淘汰学习较慢配置的风险。如果某个配置需要较长的预热期才能发挥作用，它可能在早停阶段就被误杀。缓解方法：适当增加初始预算（`min_epochs` / `min_resources`）让每个配置至少有显示潜力的机会。

## 常见误区

1. **早停=训练完成**：早停只是训练流程中的一步，停止后应做完整的模型评估，检查是否真的最优
2. **patience 越大越好**：过大的 patience 让早停失去意义，模型可能已经过拟合很远才被拉回
3. **所有任务用同一个 patience**：不同任务 loss 曲线的波动特性不同，patience 需要对应调整
4. **忽视 min_delta**：不设 min_delta 时，曲线微小波动都会重置 counter，导致停不下来
5. **用训练 loss 做早停**：训练 loss 持续下降是正常的，早停只能用验证集指标
6. **早停前不做小数据过拟合验证**：如果模型本身有 bug 根本学不会，早停只是加速暴露而已
