# 批大小与泛化

## 是什么

批大小（batch size）不仅影响训练速度和显存占用，更深刻影响模型的泛化能力——即模型在未见数据上的表现。这是深度学习中最重要的经验发现之一。

核心观察：
- **小批次（32-256）**：梯度更新含噪声，倾向于收敛到平坦极小值（flat minima），泛化通常更好
- **大批次（1024+）**：梯度估计更精确，但倾向于收敛到尖锐极小值（sharp minima），泛化可能更差
- **极端大批次（数万以上）**：泛化显著下降，除非配合特殊的训练技巧

Hochreiter & Schmidhuber (1997) 的平坦极小值理论为此提供了理论基础：平坦极小值对参数扰动不敏感，在测试集上更稳定。Keskar et al. (2016) 的实验系统性地展示了大批次收敛到尖锐极小值的现象。

## 为什么重要

理解批大小与泛化的关系直接影响模型最终效果：

- **性能上限的决定因素**：在 CV 分类任务中，batch size 从 256 增加到 4096，Top-1 准确率可能下降 1-3%（Goyal et al., 2017）。这在竞赛和生产环境中可能是质的差距。
- **调参的核心权衡**：批大小本质上是显存—速度—泛化的三角权衡。不能简单认为 "越大越好" 或 "越小越好"。
- **解释业界经验**：为什么 [[../batch-size/batch-size]] 中 炫光 说 "batch 太大网络性能会降低，有时减小 batch 有奇效"——本质上是在利用噪声注入的正则化效应。
- **领域差异的根源**：同一批大小在不同任务中效果截然不同。对比学习需要大批次（否则坍塌），推荐系统 512 封顶，分类任务 256 附近最优。参见 [[../batch-size/batch-size]] 中 BBuf 的观点。

## 如何使用

### 选择策略

**默认起点：**
- CV 分类：256-512
- NLP 预训练：512-4096（取决于模型大小）
- 对比学习：4096 以上（SimCLR 默认 8192）
- 推荐系统：32-512
- PPO/RLHF：越大越好

**验证泛化变化：**
- 增加 batch size 后观察验证集指标：若下降，尝试减小 batch size 或同步调整学习率（参见 [[./batch-size-lr-scaling]]）
- 在固定计算预算下比较：相同训练步数时大批次更稳定，但相同 epoch 数时小批次泛化可能更好

### 噪声注入的正则化视角

小批次梯度噪声可视为隐式正则化：
- 小批次（32-64）的梯度估计带有各向异性噪声，等价于在 SGD 更新上额外施加噪声扰动
- 这种噪声有助于逃离尖锐局部极小值，但也可能减慢收敛速度
- 复杂任务（高维度、多模态）中噪声可能有害，参见 [[../batch-size/batch-size]] 中 白小鱼 的观点

### 大批次优化的实用技巧

当业务需要大批次（吞吐优先）但担心泛化下降时：
- 线性学习率缩放 + warmup（参见 [[./batch-size-lr-scaling]]）
- 使用 LayerNorm / GroupNorm 替代 BatchNorm，消除 BN 统计量偏差
- 大批次 + 小学习率的组合往往优于大批次 + 大学习率
- 用梯度累积等效扩大批次（参见 [[./gradient-accumulation]]），但注意 BN 问题

### 何时坚持大批次

- 对比学习/表示学习：小批次直接导致模型坍塌
- PPO/RLHF：梯度噪声敏感的强化学习场景
- 分布式训练吞吐优化：大批次最大化计算效率

### 何时考虑减小批次

- 验证集指标随 batch size 增大持续下降
- 模型过拟合不严重但泛化明显较差
- 显存允许但模型性能异常
- 迁移开源配置到自己的数据时（参见 [[../batch-size/batch-size]] 中 BugBuster喵 的警告——"开源 repo 的默认配置往往是为了跑通 demo"）

## 常见误区

- **"批大小越大越好"**：只在一定范围内成立。跨度太大时（如 256→8192）泛化通常下降，除非配合学习率调整和 warmup。
- **"小批次就是更好"**：不绝对。复杂任务中梯度噪声可能弊大于利，且小批次 wall-clock 速度慢。
- **"大 batch 更容易过拟合"**：实际观察中大批次更易欠拟合——收敛到尖锐极小值并不意味着训练 loss 更低。
- **"泛化差异仅由批大小引起"**：学习率、学习率调度、权重初始化、正则化强度等都与批大小耦合，单独看批大小是片面的。
- **"微调时 batch size 必须和预训练一致"**：微调通常使用更小批次（因为下游任务更简单），且 BN 冻结后批大小影响更小。
