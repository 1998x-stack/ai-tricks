# 批大小与学习率缩放

## 是什么

批大小与学习率之间存在缩放关系（Scaling Law）：当批大小增大时，学习率也应当随之增大，以保持训练动力学特性一致。这一关系最早在 AlexNet（Krizhevsky, 2012）中被经验观察到，之后由 Goyal et al.（2017）在 *Accurate, Large Minibatch SGD* 中系统化为 **线性缩放规则（Linear Scaling Rule）**。

### 线性缩放规则

当批大小乘以 k 时，学习率也乘以 k，前提是：
- 学习率不超过某个上限（否则训练发散）
- 配合 warmup 策略（从较小学习率逐渐上升到目标 lr）
- 适用于 SGD / SGD with Momentum

公式：`lr_new = lr_old × (batch_size_new / batch_size_old)`

常用锚点：batch size 64 → lr 0.1，batch size 256 → lr 0.4（参见 [[../batch-size/batch-size]] 中 四点半的理一 的实践经验）。注意这个比例不是固定的，但提供了一个调参起点。

### DeepSeek Scaling Law 公式

DeepSeek 在 LLM 预训练实验中得出了更精确的幂律公式（参见 [[../batch-size/batch-size]] 中 Sam聊算法 的记录）：

```
η_opt = 0.3118 · C^(-0.1250)
B_opt = 0.2920 · C^(0.3271)
```

其中 C 为计算预算（FLOPs），η_opt 为最优学习率，B_opt 为最优批大小。幂律形式揭示了三个重要结论：
1. 计算量增大 → 最优批大小上升（C^(0.3271)）
2. 计算量增大 → 最优学习率下降（C^(-0.1250)，下降非常缓慢）
3. 多数其他超参在固定计算量下可保持不变，只需调整学习率和批大小

## 为什么重要

- **防止盲目调参**：batch size 从 256 增大到 1024 时若学习率不变，每步参数更新看到 4 倍样本但更新幅度相同，实质上是欠拟合。线性缩放规则解决了这一不匹配。
- **大规模训练的必备工具**：在 A100/H100 集群上采用大批次（数万）时，不调整学习率会导致训练效率严重下降。Warmup + 线性缩放使大批次训练成为可能。
- **超参搜索降维**：DeepSeek 表明固定计算预算下只需调整学习率和批大小，这将对二维调参转化为一维搜索。

## 如何使用

### 标准操作流程

1. 在小批次（如 256）上通过 lr range test 找到较优学习率（如 0.1）
2. 将批次扩展到目标大小（如 1024，扩大 4 倍）
3. 学习率线性缩放：0.1 × (1024/256) = 0.4
4. 使用 warmup 策略：前 5-10% 训练步数从 0 线性上升到 0.4
5. 验证 loss 曲线是否匹配小批次版本的下降轨迹

### Warmup 的必要性

大批次训练初期梯度方向不准确，过大的学习率直接导致发散：
- **线性 warmup**：从 0 线性升至目标 lr，最常用
- **指数 warmup**：从极小 lr 指数升至目标 lr
- **建议比例**：大批次（4096+）warmup 步数占总步数 5-10%；超大批次（数万+）可能需要更长

### 何时调整缩放比例

线性缩放并非普适：
- **非 SGD 优化器（Adam/AdamW）**：Adam 每参数有自适应学习率，批大小变化的影响已被部分吸收。经验上批量翻倍时 Adam 只需微调 lr（通常不动或 1.5 倍，而非 2 倍）。
- **Batch size 差距过大**：64 → 8192（128 倍），线性缩放给出的 lr 过大，需要 sub-linear 缩放或更长的 warmup。
- **不同任务特性**：对比学习使用大批次时学习率调整幅度通常小于线性缩放。

### DeepSeek 公式的实践意义

对于 LLM 预训练，给定计算预算 C：
1. 用幂律公式计算 B_opt 和 η_opt
2. B_opt 决定需要的 GPU 数量和梯度累积步数
3. 如果显存无法满足 B_opt，用梯度累积等效扩大（参见 [[gradient-accumulation]]）
4. η_opt 随 C 增加下降极慢（C^(-0.125)），意味着大规模训练中学习率基本稳定，主要调整 batch size

## 常见误区

- **"批量翻倍，学习率必翻倍"**：线性缩放适用于 SGD，不直接适用于 Adam。Adam 每参数有自适应 lr，批量变化的影响已被部分吸收。
- **"warmup 可有可无"**：大批次（1024+）不 warmup 直接上目标 lr，前几步就可能发散。
- **"缩放是双向对称的"**：batch 翻倍时 lr 可翻倍，但 batch 减半时不建议 lr 等比减半——减半后每步样本太少，仅靠缩小 lr 无法弥补信息量的损失。
- **"分布式训练中自动处理"**：DDP 默认做梯度平均，与 loss 除以累积步数配合时需要仔细确认梯度量级是否正确，避免意外的梯度缩放。
- **"DeepSeek 公式通用"**：该公式基于 DeepSeek 的具体实验配置，模型架构、数据分布、tokenizer 差异会导致系数偏移。幂律形式是普适的，具体系数需要在自己任务上重新拟合。
