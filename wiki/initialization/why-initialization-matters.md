# 初始化为什么重要

## 是什么

参数初始化是深度学习训练中看似不起眼却决定成败的一步。初始权重的设置直接影响梯度传播质量、收敛速度和最终泛化性能。从数学上看，深度网络是矩阵乘法和非线性变换的复合函数 $f(x) = \phi_L(W_L \phi_{L-1}(W_{L-1} \cdots \phi_1(W_1 x)))$。初始化决定了这个复合函数的起点位置 — 是靠近损失函数盆地的有利位置，还是卡在高原或鞍点附近的不利位置。

一个反直觉的事实是：**初始化对训练的影响往往比优化器（SGD vs Adam）和学习率更大**。这是因为初始化决定了训练开始时梯度的质量和信号传播的效率，这是后续所有优化步骤的基础。

## 为什么重要

### 梯度消失与梯度爆炸

这是初始化最重要的理论动机。对于一个没有归一化层（BN/LN）的 $L$ 层网络，前向传播为 $x^{(l)} = W^{(l)} \phi(x^{(l-1)})$。如果每层权重的方差 $\sigma^2$ 不合适，经过 $L$ 层后：

- 若 $n_{in}\sigma^2 > 1$：激活值指数级增长 → 数值溢出 → 梯度爆炸（NaN loss）
- 若 $n_{in}\sigma^2 < 1$：激活值指数级衰减 → 所有输入映射到相似值 → 梯度消失（loss 不下降）

这是历史上深度网络难以训练的根本原因 — 不是优化算法不够好，而是初始化让网络一开始就处于信号无法传播的状态。在 LSTM 和残差连接被广泛使用之前，这也是 RNN 训练的核心瓶颈。

### 收敛速度

合适的初始化可以让 loss 从训练的第一步就开始下降。不合适的初始化可能需要数百个 epoch 才能从错误的起点挣扎出来：

- Xavier 初始化相比全零初始化，在 tanh 网络上可以将收敛速度提升 10 倍以上。
- 真实案例：仅将 CNN 的初始化从默认改为 Xavier，准确率从 70% 跳到 98% — 差距远超任何超参调优的影响范围。详见 [[initialization]] 中的"初始化对 CNN 的巨大影响"。
- 初始化不当还会造成优化器需要不断"修正"权重方向，让 Adam 等自适应优化器的优势也无法发挥。

### 局部最优与鞍点

深度网络的损失面在高维空间中鞍点远多于局部极小值。初始化决定了模型落入哪个吸引域。更危险的是：

- **对称权重初始化**（如全零初始化）会导致所有神经元学到完全相同的特征，网络的有效容量退化为单神经元。这是多层网络中必须避免的陷阱 — 虽然偏置可以为零，权重必须有对称破缺。
- 线性层和 Embedding 层对初始化异常敏感：Embedding 初始化不当可能导致词向量的语义结构从起点就是扭曲的，后续训练极难纠正。
- 实验表明，从不同初始化起点出发，最终收敛到的局部最优的质量差异可达 20%+ 的测试精度。

### 对深层架构的连锁效应

初始化问题在深层网络中会层层放大：
- 第 1 层偏移 1% → 第 10 层偏移可能达 1000%（指数级放大）
- 初始化不当会让 Batch Normalization 承受巨大负担：激活值方差过大时 BN 的均值和方差估计不稳定，mini-batch 之间的抖动加剧
- 学习率的可调范围也受初始化影响：初始化差意味着学习率需要极低以避免发散，但低学习率又导致收敛极慢，形成恶性循环

不同激活函数对初始化的敏感度差异参见 [[../activation/activation-selection]]，ReLU 网络的 He 初始化推导参见 [[../activation/relu]]。

### 现代网络的初始化依赖

即使有了 LayerNorm、残差连接和 Attention，现代 Transformer 对初始化仍然敏感：
- DeepSeek 等大模型固定初始化标准差为 0.06，这一值是经过大量实验得到的经验结果
- Transformer 中 FFN 层的初始化与 Attention 输出层的初始化需要不同的缩放策略
- Pre-LN 和 Post-LN 结构对初始化的容忍度也不同 — Pre-LN 对初始化更鲁棒，而 Post-LN 需要更精细地调参

详见 [[../../contradictory]] 中关于现代架构初始化争议的讨论。

## 如何使用

### 初始化选择速查

| 场景 | 推荐初始化 | 原因 |
|------|-----------|------|
| ReLU 激活 | He/Kaiming Uniform | 补偿 ReLU 方差损失 |
| Tanh/Sigmoid 激活 | Xavier/Glorot Uniform | 方差守恒假设成立 |
| Embedding 层 | 截断正态分布 (std=0.02) | 避免尾部异常值 |
| RNN/LSTM | 正交初始化 | 保持信号范数 |
| 无激活函数层 | Xavier 或 $1/\sqrt{n}$ | 线性方差传播 |
| 大模型预训练 | Normal (std=0.06) | DeepSeek 经验值 |
| Bias | 全零 | 无对称破缺需求 |

### 实践建议

1. **优先使用框架默认**：PyTorch `nn.Linear`/`nn.Conv2d` 默认 Kaiming Uniform，对大部分场景足够。
2. **匹配激活函数**：根据上表匹配激活函数与初始化方法。
3. **有预训练时用预训练**：预训练权重是比任何初始化更好的起点。加载后务必验证（`strict=True`）。
4. **新架构先过拟合再调初始化**：先不加 BN/Dropout，确保模型 capacity 能过拟合一个 batch，再系统调整初始化和正则化。
5. **异常排查流程**：
   - Loss NaN → 初始化方差过大 → 减小 $\sigma$ 或换用 Xavier/He
   - Loss 不下降 → 检查激活函数与初始化是否匹配
   - Loss 下降极慢 → 初始化方差过小或激活函数不匹配 → 检查 $n\sigma^2$ 是否接近 1

## 常见误区

1. **"默认初始化就够用，不用管"**。PyTorch 默认确实是 Kaiming Uniform，但只对 ReLU 最优。换用 tanh/sigmoid 后需要手动切换 Xavier。跨框架复现时尤其注意 — PyTorch 和 TensorFlow 的默认初始化不同（见 [[initialization]] 中的框架差异讨论）。
2. **"初始化只影响收敛速度，不影响最终精度"**。错误。真实案例表明不同初始化可以让同一模型从 70% 到 98% — 差距远超超参调优范围。
3. **"初始化方法可以随意组合"**。不同层的初始化需要与激活函数协同。ReLU 层用 Xavier 初始化，信号每层衰减一半，100 层后信号消失。
4. **"长训练可以弥补错误初始化"**。部分可以，但成本极高。某些错误（如方差过大）会导致训练早期发散，再长也无法恢复。也有记录表明长训练中换初始化可以跳出局部最优 — 见 [[initialization]] 中相关技巧。
5. **"LayerNorm/BN 可以解决所有初始化问题"**。归一化层缓解的是训练过程中的分布偏移，无法修复跨层的指数级方差传播。初始化和归一化是互补的两层防线，不是替代关系。
6. **"均匀分布一定比正态分布好"**（或反之）。两种分布形态的差异在实际中通常可以忽略，关键是正确的方差缩放因子，而不是分布形态。
