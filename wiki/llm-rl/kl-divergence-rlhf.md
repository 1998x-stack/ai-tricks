# KL 散度在 RLHF 中的应用

## 是什么

KL 散度（Kullback-Leibler Divergence）在 RLHF 中作为策略约束，衡量当前策略 π_θ 与参考策略 π_ref（通常是 SFT 模型）之间的分布差异。在 PPO-RLHF 中，KL 散度以惩罚项的形式加入奖励函数：

`R_total = R_RM - β * KL(π_θ || π_ref)`

其中 β 是 KL 惩罚系数。

### 为什么需要 KL 约束

RLHF 中的 KL 约束并非锦上添花，而是**训练能否成功的关键**：

1. **防止模式坍塌（Mode Collapse）**：没有 KL 约束，策略会快速收敛到 RM 打分最高的少数几类回复，失去多样性。极端情况下模型变成"复读机"。
2. **防止灾难性遗忘**：KL 约束让策略不偏离 SFT 过远，保留预训练和 SFT 阶段学到的语言能力和知识。参见 [[../llm-sft/llm-sft]]。
3. **稳定训练**：KL 约束充当了 natural gradient 的角色，平滑了优化轨迹，让 reward 的噪声信号不至于直接颠覆策略。
4. **保持探索能力**：通过约束分布偏移，模型仍然能生成多样化的回应，而非过早收敛到局部最优。

### 与其他约束方式的对比

| 方式 | 机制 | 特点 |
|------|------|------|
| **KL 惩罚** | 奖励函数中减去 β·KL | 连续、可微、自然适配 RL 框架 |
| **PPO Clip** | 限幅策略比率的范围 | 单步更新幅度限制，与 KL 互补 |
| **Entropy Bonus** | 奖励中加上熵项 | 鼓励探索，防止同质化 |
| **SFT 数据混合** | Prompt 池混入通用 SFT 数据 | 从数据层面锚定基础能力 |

KL 惩罚 + PPO Clip 是双重保护：Clip 控制单步比率，KL 控制全局分布偏移。

## 为什么重要

### KL 是"安全带"还是"紧箍咒"？

KL 散度在 RLHF 中的角色是双重性的：

- **太小（β 过小）**：安全带太松，策略可以随意漂移。Reward 可能快速上涨但模型迅速过拟合到 RM 的偏见上，生成质量下降。KL 爆炸是典型信号。
- **太大（β 过大）**：紧箍咒太紧，策略几乎被冻结。Reward 停滞或增长极为缓慢，模型无法有效学习偏好。这就是 KL 停滞。

找到合适的 β 就是在"允许学习"和"防止乱学"之间寻找平衡点。这个平衡点取决于：SFT 质量、偏好数据量、RM 的准确性。

### KL 不会自然持续下降

一个常见误解是"KL 应该持续下降才正常"。实际上，在 RL 训练中 KL 通常先上升（策略从 SFT 出发探索偏好空间），然后趋于平稳。如果 KL 持续为 0 或接近 0，说明训练没有生效。

## 如何使用

### β 的调节策略

**初始值：从 0.001 开始**

对于大多数 RLHF 场景，推荐从 0.001 开始尝试。原因：

- 0.001 足够小，让 KL 约束松到"允许模型开始学习"
- 不会在开局就束缚死策略
- 后续根据训练动态逐步调整

**调节逻辑**：

| 现象 | 诊断 | 操作 |
|------|------|------|
| Reward 上升且 KL 缓慢上升 | 正常训练 | 维持当前 β |
| Reward 上升但 KL 爆炸增长 | KL 惩罚不足 | 增大 β（如 ×2-5） |
| KL 很低，Reward 不涨或极慢 | KL 惩罚过强 | 减小 β（如 ×0.2-0.5） |
| KL 很低，Reward 也在涨 | 可能已 hacking | 检查生成质量 |
| KL 震荡剧烈 | β 不匹配 | 同时检查 LR 是否合适 |

### KL 爆炸的处理

KL 爆炸是训练中最危险的信号，意味着策略已经严重偏离了 SFT 边界：

1. 立即增大 β（在当前值基础上 ×5-10）
2. 如果 β 已很大（>0.1），检查 RM 是否产生了极端偏差的打分
3. 同时降低学习率（减小更新步长）
4. 考虑混入更多 SFT 数据来"锚定"模型

### KL 停滞的处理

KL 几乎为 0 且 reward 不增长：

1. 减小 β（当前值 ×0.1-0.5）
2. 检查学习率是否过低——LR 太低即使 β 合理也无法有效更新
3. 前面提到：如果基模能力上限本身就低，rollout 多个回复的 reward 都低，那 KL 怎么调都没用，需要换模型或优化 SFT

### 特殊场景：基模不做 SFT 时的 KL

对于没有经过 SFT 的基模直接做 RL，KL 散度可以不开。原因是基模本身生成多样性好于 SFT 模型（SFT 后分布会变得更集中、更确定），所以基模模式下的 KL 约束反而会不必要地限制探索。参见 [[ppo-rlhf]]。

### KL 与熵的关系

- **KL 散度**衡量策略相对于参考模型的偏移
- **熵（Entropy）**衡量策略本身的多样性

两者互相补充但不相同：KL 低 + 熵低 = 策略停在 SFT 附近且缺乏多样性（可能是 β 过高或模型能力上限）；KL 高 + 熵低 = 策略走偏了但缺乏多样性（典型的 hacking 信号）；KL 高 + 熵高 = 策略在探索（初期正常）；KL 低 + 熵高 = 理想状态（有足够多样性但不偏离 SFT 太远）。

在训练监控中，需要同时追踪 KL 和熵两条曲线才能准确判断训练状态。

### KL 系数的自适应调节

一些方案提出自适应调节 KL 系数：设定目标 KL 值区间（如 0.01-0.1），当 KL 超过上限时自动增大 β，低于下限时自动减小 β。这种方法可以减少手动干预，但需要谨慎设置目标区间（太保守则学不到东西，太激进则失去保护意义）。

## 常见误区

1. **KL 越小越好**：KL 为 0 意味着模型完全没有学习，RL 训练没有发生。合理的 KL 应该是一个非零的稳定值。
2. **β 设好了就不用管**：KL 惩罚系数可能需要随训练阶段调整。初期可能需要较松的约束以便学习，后期可能需要收紧来防止过度偏离。
3. **只看 KL 不看 reward 曲线**：KL 和 reward 需要结合分析。单独看 KL 无法判断训练是否健康。
4. **KL 惩罚和 Entropy Bonus 是一回事**：两者截然不同。KL 约束策略偏离参考模型；Entropy Bonus 鼓励策略自身多样化。需要同时关注。
5. **所有场景的 β 通用**：不同模型大小、不同数据量、不同 RM 质量下的最优 β 差异很大。必须针对具体训练场景调节。
