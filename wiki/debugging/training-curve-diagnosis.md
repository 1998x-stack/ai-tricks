# 训练曲线诊断

## 是什么

训练曲线诊断是通过观察 loss（和评估指标）随时间/step 的变化形态来推断训练状态的技术。训练曲线是调试过程中最直观但最容易被误读的信号——不同曲线形态对应不同的根因，而正确的诊断能大幅缩小排查范围。

核心原则：**loss 是比准确率更可靠的信号**。准确率可能长期为 0 然后突然跳到 1（在分类问题中常见），但 loss 提供连续的、稳定的变化信号。不要因为前期准确率低就提前终止训练。

---

## 为什么重要

训练曲线告诉你"模型当前卡在哪里"，这是调参决策的基础。未经诊断的调参是盲调——你既不知道问题在哪，也无法判断改动是否有效。

| 诊断结果 | 反映的问题 | 优先级 |
|----------|-----------|--------|
| train loss 高 + val loss 高 | 欠拟合，模型容量或学习率不足 | 最高（blocker） |
| train loss 低 + val loss 高 | 过拟合，泛化差距大 | 中 |
| train loss 低 + val loss 低 | 训练正常 | 检查指标是否达标 |
| train/val 差距正常但指标不达标 | 任务/数据问题 | 换方向 |

## 如何使用

### 常见曲线形态解读

#### 形态一：Loss 稳步下降后持平

训练 loss 和验证 loss 都稳定下降，最终在某个值附近平稳。训练/验证 gap 小且稳定。

- **含义**：训练正常。如果平台期的 loss 高于你期望的值，是欠拟合信号——容量或训练时间不够。
- **处理**：继续训练，增大模型容量，或检查是否已经达到数据贝叶斯下限。

#### 形态二：Loss 持续下降但验证集不降

train loss 持续下降，val loss 先降后升或不再下降，gap 随时间扩大。

- **含义**：**过拟合**。模型开始记忆训练数据而非泛化。
- **处理**：加正则化（Dropout、Weight Decay），或早停。详见 [[../regularization/overfitting-underfitting]]。

#### 形态三：Loss 震荡剧烈

loss 曲线锯齿状，每步之间跳跃剧烈。

- **含义**：学习率太高或 batch size 太小。小幅震荡是 SGD 的正常特征，但振幅超过 loss 本身的 20-30% 就值得关注。
- **处理**：降低 lr，或增大 batch size。同时检查数据增强是否引入了高方差噪声。

#### 形态四：Loss 先快速下降后停滞（平台期）

前期下降很快（几个 step 内 loss 从 4.0 降到 1.0），然后突然停滞，loss 几乎不再变化。

- **含义**：模型学到了一些容易的粗粒度特征（如分类任务中学到了类别先验分布），但无法学到细粒度区分。"卡在局部最优"或"特征瓶颈"。
- **处理**：检查是否需要更深的网络、是否特征提取器已经饱和、是否 loss 函数需要调整（如 Focal Loss 或重新加权）。

#### 形态五：Loss 先降后升到 NaN

loss 先正常下降，然后突然开始上升，最终变成 NaN（见 [[../debugging/nan-diagnosis]]）。

- **含义**：渐变型 NaN，梯度爆炸。
- **处理**：降低学习率、添加 warmup、检查是否存在梯度累积溢出。

#### 形态六：Loss 完全不动

loss 从始至终在一个固定值附近，微小波动但不下降。

- **含义**：参数未更新、学习率为零、或模型处于不学习状态。
- **处理**：检查 optimizer 是否包含了所有模型参数、lr 是否为有效值、loss 是否连接到了计算图。

#### 形态七：Loss 爆炸式上升

loss 在某个点突然跳升几个数量级（如 0.1 跳到 1000），然后有可能降回来。

- **含义**：遇到了极端坏样本，或梯度裁剪把梯度方向彻底扭曲。
- **处理**：检查那个 step 的输入数据。移除或屏蔽该样本。同时检查裁剪阈值是否合理（见 [[../debugging/gradient-clipping]]）。

### 训练/验证差距解读

监控 train loss 与 val loss 的**差值**比看绝对值更有意义：

- **gap < 5%**：正常训练，模型在有效泛化。
- **gap 5-15%**：轻微过拟合，监控变化趋势。如果 gap 仍在增大，准备加正则化。
- **gap > 15%**：严重过拟合。数据量不足或模型容量远超所需。优先级：加数据 > 加正则化 > 降模型容量。
- **gap 为负数（train > val）**：特殊场景，通常意味着验证集分布比训练集容易，或者验证集样本太少导致统计不稳定。常见于小验证集或验证集与训练集来自不同分布。

### 早停的判断

不要在大 gap 出现的第一时间就早停。有些模型的泛化差距会在训练后期自然缩小。一个可靠的早停策略：

1. 先确保模型能在训练集上充分过拟合（见 [[../debugging/small-data-overfit-test]]）。
2. 记录 val loss 的最优值。
3. 设置 patience（如 10-20 个 epoch）：val loss 连续 patience 轮没有突破最优值才停止。
4. 恢复最优 checkpoint 而非最后一步。

## 常见误区

- **"准确率是比 loss 更好的监控指标"**。准确率在分类任务中是离散的（0 或 1），可能长期不变然后突变。loss 提供连续信号，能更早地发现训练异常。训练过程中应优先观察 loss，用准确率做最终评估。
- **"train loss 降得很低就是好模型"**。低 train loss 只是必要条件。如果没有同时关注 val loss，可能只是过拟合到了训练数据。配合验证集和测试集来确认。
- **"loss 震荡大就是有问题"**。SGD 的 loss 曲线天然有噪声。振幅在合理范围（loss 的 10-20%）内且不持续增长，属于正常现象。但如果使用了 Adam 等自适应优化器仍然大幅震荡，需要注意。
- **"两个曲线的 gap 一定要小"**。gap 的大小取决于任务难度和数据量。对于高噪声任务（如医疗影像、用户行为预测），gap 15-20% 也是正常的。关键看 gap 是否在持续扩大而非绝对值大小。
- **"曲线波动是因为学习率衰减策略太复杂"**。先用最简单的线性衰减或步长衰减（step decay），确认模型和基础调参到位后再尝试 cosine、one-cycle 等高级策略。复杂调度引入的额外曲线形态容易掩盖真正的训练问题。
