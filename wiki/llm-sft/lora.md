# LoRA 低秩适配

## 是什么

LoRA（Low-Rank Adaptation）是一种参数高效微调方法，核心思想是在预训练权重矩阵旁插入可训练的低秩分解矩阵，从而在不修改原始权重的前提下近似全量微调的参数更新量。

对于一个预训练权重矩阵 $W \in \mathbb{R}^{d \times k}$，LoRA 将其更新量 $\Delta W$ 分解为两个低秩矩阵的乘积：

$$\Delta W = BA, \quad B \in \mathbb{R}^{d \times r}, A \in \mathbb{R}^{r \times k}, \quad r \ll \min(d, k)$$

前向传播变为 $h = Wx + BAx$，训练时只更新 $A$ 和 $B$，$W$ 被冻结。训练完成后可以将 $BA$ 合并回 $W$（即 $W' = W + BA$），推理时零额外延迟。

LoRA 的核心参数包括：

- **Rank (r)**：低秩矩阵的秩，控制可训练参数量。r 越大，表达能力越强，但参数量也越大。常见值 8、16、32。
- **Alpha (α)**：缩放系数，输出缩放为 $\frac{\alpha}{r}$。影响 LoRA 分支的更新幅度。常见值 16、32、64。
- **Target Modules**：应用 LoRA 的目标模块列表，如 `q_proj, v_proj` 或全 `attention` 模块。
- **Dropout**：LoRA 层的 dropout 比率，用于缓解过拟合。常见范围 0~0.1。
- **学习率**：LoRA 的 LR 通常为 1e-4~3e-4，高于全量微调的 1e-5~5e-5。

## 为什么重要

LoRA 解决了大模型微调的核心矛盾：全量微调 7B+ 模型需要数十 GB 显存，普通硬件无法承受；而冻结全部权重、只训练小型适配器，可以在单卡消费级 GPU 上完成微调。

具体优势：

- **显存大幅降低**：可训练参数量从 100% 降至 0.1%~1%。以 LLaMA-7B 为例，全量微调需约 112GB 显存，LoRA (r=8) 仅需约 16GB。
- **快速切换任务**：不同任务的 LoRA 权重可以独立保存和加载，基座模型只需一份。服务端可动态切换 adapter，实现多任务共用一个基模。
- **防止灾难性遗忘**：由于原始权重被冻结，模型的基础能力几乎不受影响。这在数据量少时尤为重要。
- **推理零开销**：LoRA 权重可以合并入原始权重，推理时无额外计算。

## 如何使用

### Rank 选择

Rank 控制 LoRA 的表达能力与参数量之间的权衡：

- **数据量多（>10k 条）或任务复杂**：r=32 或 r=64，给模型足够的自由度去拟合复杂模式。
- **数据量少（<1k 条）或任务简单**：r=8 或 r=16。数据少时大 rank 极易过拟合——模型有太多参数去"记住"训练样本而非学习范式。
- **通用起点**：大多数业务任务 r=8 或 r=16 已足够。从 r=8 开始，如果欠拟合（训练 loss 降不下去），增大 rank。

Rank 不是越大越好。过量 rank 在数据有限时表现为验证集效果骤降——模型记住了每条样本的确切格式，失去泛化能力。

### Alpha 选择

Alpha 控制 LoRA 分支的更新幅度。常见规则：

- **Alpha = rank × 2** 是一个常见默认值（rank=8 -> alpha=16, rank=16 -> alpha=32）。
- Alpha 过小导致 LoRA 分支更新过弱，模型"学不动"；Alpha 过大可能导致训练初期 loss 震荡。
- 实际使用中 Alpha 的敏感度低于 rank 和 LR。从 16 或 32 起步即可。

### Target Modules

在 Transformer 架构中，常见的 LoRA 目标模块选择策略：

- **仅 Q 和 V**（`q_proj, v_proj`）是最经典配置。QLoRA 论文发现 Q+V 足以在大多数任务上达到接近全量微调的效果。
- **全 Attention**（`q_proj, k_proj, v_proj, o_proj`）提供更强的表达能力，约 2x 参数量，收益在某些任务上有限。
- **扩展至 FFN**（`gate_proj, up_proj, down_proj`）进一步提升模型容量，但参数量增加更多。
- 从 `q_proj, v_proj` 起步，如发现欠拟合再逐步扩展至全 Attention 和 FFN。

### Dropout

LoRA dropout 用于缓解小数据场景的过拟合：

- **数据量大（>10k）**：dropout=0 或 0.05。
- **数据量小（<1k）**：dropout=0.1。
- 过高的 dropout（>0.2）可能严重阻碍 LoRA 适配器的学习，除非数据极其有限。

### 学习率

LoRA 的 LR 范围与全量微调不同，因为只有少量参数被更新：

- **常见范围**：1e-4 到 3e-4，从 2e-4 起步是一个合理选择。
- **小数据、强基座模型**：可以用更低 LR（5e-5 到 1e-4），防止在小数据上过拟合。
- **LR 从大到小调**：从 3e-4 开始，如果 loss 震荡则降至 2e-4、1e-4。
- 对比：全量微调通常用 1e-5~5e-5（低一个数量级），因为所有参数都在更新，需要更小步长保护预训练权重。

## 常见误区

### "Rank 越大效果越好"

这是最常见的 LoRA 误区。Rank 增加参数量，在小数据场景下直接导致过拟合。Rank 的选择应匹配数据规模和任务复杂度。先小后大，从 8 或 16 起步。

### "LoRA 可以完全替代全量微调"

LoRA 在多数任务上逼近全量微调效果，但在以下场景仍有显著差距：需要学习全新知识或强领域偏移的任务、需要极大改变模型行为模式的任务。LoRA 的本质是在预训练权重附近搜索最优解，无法跳出这个局部邻域。

### "Target modules 越多越好"

盲目增加目标模块不一定带来收益。Q+V 已覆盖大部分任务需求，扩展至全 Attention 或 FFN 收益递减。增加的参数量意味着更大的显存消耗和过拟合风险。

### "Alpha 要保持和 rank 相等"

Alpha 不需要等于 rank。两者是独立参数，控制不同的东西：rank 控制参数量和表达能力，Alpha 控制更新幅度。常见做法是 Alpha 设为 rank 的 2 倍。

### "LoRA 训练完后可以不合并权重"

推理时如果不合并权重，会带来额外的计算延迟（每次前向传播需计算 BAx）。除非在服务端动态切换 adapter 的场景，否则建议合并权重后部署。
