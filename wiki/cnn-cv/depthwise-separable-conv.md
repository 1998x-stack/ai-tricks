# 深度可分离卷积

## 是什么

深度可分离卷积（Depthwise Separable Convolution）将标准卷积分解为两个独立步骤：

**深度卷积（Depthwise Convolution）**：每个输入通道独立使用一个卷积核进行空间卷积。输入 C 个通道使用 C 个独立的 3×3 卷积核，每个核只在一个通道上操作，输出 C 个特征图。完成"空间特征提取"，没有跨通道的信息交互。

**逐点卷积（Pointwise Convolution）**：使用 1×1 卷积在通道维度上对深度卷积输出进行线性组合。F 个 1×1 卷积核将 C 个通道映射到 F 个输出通道。完成"跨通道特征融合"。

标准卷积 `(k×k×C×F)` 分解为深度卷积 `(k×k×1×C)` 加逐点卷积 `(1×1×C×F)`。这是 MobileNet 系列的核心构建块，也是 Xception、EfficientNet-Lite 等轻量级网络的基础。从数学上看，深度可分离卷积将空间相关性和通道相关性解耦，使两者可用更少的参数独立建模。

**参数与 FLOPs 对比**：

| 卷积类型 | 参数量 | FLOPs | 以 3×3, C=64, F=128 为例 |
|----------|--------|-------|--------------------------|
| 标准卷积 | k²CF | k²CFHW | 73,728 params |
| 深度可分离 | k²C + CF | (k²C + CF)HW | 8,768 params |
| 压缩比 | — | — | ~8.4× |

标准卷积参数量与通道数呈二次关系（O(C×F)），可分离卷积为线性（O(C+F)）。这一差异随通道数增长急剧放大。当 C=F=512 时，标准卷积参数为 9×512² ≈ 2.36M，可分离卷积仅为 9×512 + 512² ≈ 267K。深层网络中通道数可达 1024 甚至 2048，参数差异可达到数十倍。

**MobileNet 系列演进**：

| 版本 | 核心创新 | ImageNet 精度 | 计算量 |
|------|----------|---------------|--------|
| V1 | DW+PW 替代标准卷积 | 70.6% | 569M MFLOPs |
| V2 | 倒残差 + 线性瓶颈 | 72.0% | 300M MFLOPs |
| V3 | NAS + h-swish + SE | 75.2% | 219M MFLOPs |

- **MobileNetV1**：简单替换所有 3×3 标准卷积（第一层除外），无 shortcut。参数量降低约 85%，精度仅损失约 1%。是深度可分离卷积的奠基性工程验证。
- **MobileNetV2**：倒残差（Inverted Residual）——先 1×1 扩展通道，再 3×3 depthwise，最后 1×1 投影压缩。瓶颈处使用 shortcut，激活函数仅用于扩展层（输出层用线性瓶颈避免 ReLU 信息丢失）。
- **MobileNetV3**：NAS + NetAdapt 搜索最优结构，引入 h-swish 激活（ReLU6 的变体，计算更高效）和 SE 模块（对 depthwise 输出做通道注意力），在更少 FLOPs 下实现更高精度。

## 为什么重要

**计算效率显著提升**：MobileNetV1 参数量降低约 85%，仅损失约 1% 精度。这种效率使 CNN 可部署在手机、嵌入式设备和边缘节点。MobileNetV2 以约 50ms 移动端推理延迟在精度上超过 GoogleNet 约 3 个百分点。

**参数线性增长**：`[[../cnn-cv/cnn-cv]]` 指出："替代完后考虑给替代可分离的通道数乘以 2，因为可分离卷积的参数和计算量都是线性增长的。" 标准卷积翻倍通道数计算量增 4 倍，可分离卷积仅增约 2 倍。这使深度可分离卷积在超宽网络中仍能保持计算可控。

**表达力依然充足**：深度卷积与逐点卷积解耦后各自可独立优化。倒残差在低维压缩、高维计算——传统残差在 64→64→64 的低维计算，MobileNetV2 在 64→256→64 的高维计算，用更多 FLOPs 在深度卷积阶段获得更丰富的空间特征。

**为架构搜索提供空间**：线性增长使 NAS 可在通道数维度自由探索。EfficientNet 复合缩放策略正依赖于此——同时缩放深度、宽度和分辨率时，MBConv 结构使各维度的参数保持平衡可控。

**分组卷积与通道混洗**：深度可分离是分组卷积的极端（每通道一组）。ShuffleNet 使用中间态——分组数 g 介于 2 和 C 之间，配合通道混洗实现组间通信（permute + reshape 操作 O(1) 代价）。Xception 走向另一端——极致的深度可分离（g=C）。

**GhostNet**：先做少量标准卷积生成"固有"特征，再通过线性变换（depthwise conv）生成"幽灵"特征后拼接。在相同 FLOPs 下 GhostNet 精度优于 MobileNetV3。深度可分离的本质被进一步抽象为"特征冗余利用"。

**硬件加速差异**：NVIDIA GPU 从 Turing 起对 depthwise conv 提供专门加速。Pascal 及以前架构上 memory bandwidth 瓶颈使实际速度远低于理论值（算力利用率 10-30% vs 标准卷积 60-80%）。ARM 移动端 SoC 优化程度较高。部署前必须 profiling。

## 如何使用

**替换策略**：`[[../cnn-cv/cnn-cv]]` 建议："能用可分离卷积替代的卷积一定要替代，一般除了第一个卷积，都能替代。" 第一层外的标准卷积几乎都可替换。

**通道数乘 2 策略**：`[[../cnn-cv/cnn-cv]]` 强调："替代完后考虑给替代可分离的通道数乘以 2……这样做依然有速度增益。" 原标准卷积 `9CF`（k=3），替换后 `9C + CF`，通道翻倍后 `9C + 2CF`。`9C + 2CF < 9CF` 等价于 `F > 9/7 ≈ 1.29`——实际网络中 F 远大于此。

**网络适配**：
- MobileNetV1：全替换（首层除外），无 shortcut，简单高效。
- MobileNetV2：倒残差——3×3 depthwise 在高维计算，低维 shortcut。
- Xception：全部替换 + 更多跨层连接，超越 Inception-v3。
- EfficientNet-Lite：MBConv + SE + 复合缩放。
- ConvNeXt：7×7 depthwise conv，将分离思想扩展到更大感受野。

**混用场景**：以下情况保留标准卷积——第一层（RGB 通道间相关性强）、通道极少（C < 16）、硬件未优化 depthwise（旧款 GPU/CPU）。

**训练技巧**：深度可分离收敛更慢，需更长训练步数和更大初始 LR。建议余弦退火 LR 调度。数据增强在小模型收益显著（参数量少易欠拟合）。

## 常见误区

1. **深度可分离卷积无精度损失**：替换后通常 0.5%-2% 精度下降。通道乘 2 + 训练技巧可弥补甚至反超。

2. **替换后直接获得 8 倍加速**：理论 FLOPs ≠ 延迟。深度卷积计算密度低、访存高，未优化硬件仅 2-3 倍加速。

3. **只适用于小模型**：Xception 超越 Inception-v3，EfficientNet-L2 达 SOTA——大模型上同样有效。

4. **所有卷积都应替换**：第一层 RGB 相关性高，标准卷积更好。1×1 再分解无意义。

5. **深度卷积核必须是 3×3**：5×5、7×7 depthwise 在 MobileNetV3 和 ConvNeXt 中使用。核增大后优势减弱（9C→25C→49C）。

6. **可分离卷积 = 轻量网络的唯一选择**：GhostNet（特征冗余利用）和 ShuffleNet（分组混洗）提供了不同思路。
