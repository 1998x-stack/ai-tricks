# 感受野

## 是什么

感受野（Receptive Field, RF）定义为卷积神经网络中某一层特征图上的一个像素点对应到输入图像上的区域大小。输出特征图中的一个值由输入图像中多大范围的像素计算而来，这个范围就是该层的感受野。

对于单层卷积，感受野大小等于卷积核尺寸。例如 3×3 卷积的输出特征图中每个点的感受野是 3×3。当网络层层堆叠后，深层特征图的感受野会逐层累积，覆盖输入图像上更大的区域。

感受野的计算遵循递推公式：

```
RF_l = RF_{l-1} + (kernel_size_l - 1) × stride_accum_{l-1}
```

其中 `stride_accum_{l-1}` 是第 l-1 层及之前所有层步长的乘积。这个递推关系揭示了一个关键事实：步长对感受野的影响是乘积性的——如果前两层各有 stride=2，第三层的步长贡献会被放大 4 倍。这就是为什么下采样操作会急剧加速感受野的扩张速度。

**经典网络感受野对比**：

| 网络 | 输入尺寸 | 最后一层 RF | 覆盖比例 |
|------|----------|-------------|----------|
| AlexNet | 227×227 | 195×195 | ~86% |
| VGG16 | 224×224 | 212×212 | ~95% |
| ResNet-50 | 224×224 | 427×427 | >100% |
| ResNet-152 | 224×224 | 843×843 | >100% |

VGG16 的 RF 约为 212，几乎覆盖整张输入图像；ResNet 系列由于更多下采样和层数，RF 可以远大于输入尺寸。了解网络的实际感受野数值是架构设计的基础。

**有效感受野（Effective Receptive Field, ERF）**：理论感受野假定所有像素对输出的贡献相等，但实际上 RF 中心区域的像素对输出有更大的影响，边缘区域呈高斯衰减。ERF 通常只有理论 RF 的 1/3 到 1/2。这意味着要让网络真正利用大感受野，需要比理论计算更多的层数或更大的卷积核。ReLU 截断负值后的梯度差异和随机初始化都会影响 ERF 的实际分布。

**具体网络计算示例**：

以 VGG16 为例展示累加过程。前两层：

- 第一层 3×3 conv（stride=1）：RF = 0 + (3-1)×1 = 3
- 第二层 3×3 conv（stride=1）：RF = 3 + (3-1)×1 = 5

每经过一层 stride=1 的 3×3 conv，RF 增加 2。经过 stride=2 的 pooling 后，`stride_accum` 翻倍，后续每层 RF 增长加速。所有 13 个卷积层 + 4 个 pooling 后，VGG16 理论 RF 约为 212。

**不同任务对 RF 的要求**：

| 任务 | 所需 RF 参考 | 设计原则 |
|------|-------------|----------|
| 图像分类（ImageNet） | ≥ 最大目标尺寸（~200px） | 最后一层 RF 覆盖目标 |
| 语义分割（Cityscapes） | 多尺度，ASPP 分支覆盖 3~37 | ASPP 多膨胀率组合 |
| 目标检测（COCO） | 各预测层匹配对应目标尺度 | FPN 不同层不同 RF |
| 图像去噪 | 局部 patch（~64px） | 小 RF，少下采样 |
| 超分辨率 | 局部 patch（~32px） | 极小 RF，无下采样 |

## 为什么重要

感受野决定了网络在做决策时"看到了多少上下文"，直接影响模型在不同视觉任务上的表现：

- **分类任务**：ImageNet 分类中最后一层 RF 最好覆盖整个目标甚至更大。有 FC 层的网络（如 VGG16）通过全连接层获得隐式全局信息，对 RF 依赖较小。但纯卷积网络（FCN）最后一层的 RF 必须覆盖数据集中最大的有意义物体。`[[../cnn-cv/cnn-cv]]` 强调："FCN 没有 FC 层兜底，感受野必须覆盖目标；有 FC 的网络好歹有个全局信息。"

- **语义分割**：分割是逐像素分类任务，每个像素的预测依赖足够大的上下文。DeepLab 系列使用空洞卷积（Dilated Convolution）在不下采样的情况下指数级扩大 RF。DeepLabv3+ 中 ASPP 模块使用 {1, 6, 12, 18} 的膨胀率组合，单个 3×3 膨胀卷积即可覆盖 37×37 的感受野。

- **目标检测**：检测网络为每个锚点或查询预测类别和位置。若 RF 远小于目标尺寸，网络无法"看到"完整目标导致漏检；若 RF 远大于目标又引入背景噪声。SSD 在不同层级的特征图上做预测，正是利用不同层 RF 的差异适配不同尺度目标。`[[../cnn-cv/cnn-cv]]` 指出："最后一层至少能看到最大的有意义物体；更大感受野通常无害，若超出太多网络可学出单位卷积。"

- **底层视觉（Low-level Vision）**：去噪、超分辨率等任务不需要全局上下文，RF 只需覆盖局部 patch。`[[../cnn-cv/cnn-cv]]` 指出："对于某些 low-level 的任务而言，比如图像降噪，感受野可能也就局部 patch 大小。low-level 任务感受野不用太大，下采样意义更小，应尽可能少下采样。"

- **关键点检测**：关键点定位依赖局部邻域特征，RF 过大可能引入干扰。FCN 做 landmark 检测时（如 `[[../cnn-cv/cnn-cv]]` 推荐的 heatmap 方法），RF 控制直接影响定位精度。

## 如何使用

**计算感受野**：对每一层（特别是下采样前一层和最后一层）计算 RF 并与数据集目标尺寸对比。使用 `torchsummary` 或 `receptivefield` Python 库自动计算。手动计算关键参数为前层 RF、当前核大小 k、当前步长 s、之前步长累积 S。

**分类网络设计**：最后一层 RF ≥ 数据集最大目标尺寸。`[[../cnn-cv/cnn-cv]]` 指出："最终能够下采样的最大深度，以该层的感受野以及数据集中最大的有意义物体尺寸决定。" 更大 RF 通常无害，因网络可学出单位卷积忽略多余区域。若 RF 不足，增加深度或下采样次数。

**分割网络设计**：常用空洞卷积替代最后几个下采样层（output_stride=8/16），在保持分辨率的同时扩大 RF。DeepLab 的 dilation strategy 在 backbone 最后几个 stage 使用 stride=1 的膨胀卷积。

**底层视觉网络设计**：使用较少池化或完全避免下采样（如 SRResNet）。核以 3×3 堆叠为主。U-Net 风格编码器-解码器在降噪任务中更关注局部感受野。

**下采样时机控制**：下采样应在 RF 足够大之后进行。`[[../cnn-cv/cnn-cv]]` 强调："在使用下采样之前要保证在这一层上有足够的感受野……越靠近输入层相对感受野应该越小。" 两个下采样之间的子网络必须有足够 RF 才能将空间信息编码到下一层通道。前几层高频下采样换速度，中间层降低下采样密度。

**相对感受野原则**：靠近输入层的层空间信息冗余度高，相对感受野应越小。这指导了"前几层高频下采样，中间层降低密度"的设计模式。`[[../cnn-cv/cnn-cv]]` 中相对感受野的概念——输入层附近 RF 应小，逐层增大后下采样，避免信息压缩损失。

**感受野可视化**：使用梯度加权、遮挡实验或引导反向传播等可视化方法可以直观观察网络实际使用的 RF 区域。这对于诊断网络的注意力分布和验证 RF 设计是否正确非常有帮助。

## 常见误区

1. **感受野越大越好**：对底层视觉任务，过大 RF 引入无关背景噪声。即使在高层次任务中，超大 RF 也不一定更好——网络可通过 1×1 卷积"忽略"多余区域，但浪费了参数和计算量。

2. **忽略膨胀卷积的感受野计算**：膨胀卷积有效核大小为 `k + (k-1)(rate-1)`。例如 3×3 在膨胀率 rate=2 时有效核大小为 5，RF 为 5×5，但只有 9 个非零点。RF 计算时必须使用有效核大小而非原始核大小。

3. **仅以最后一层 RF 为指标**：中间层的 RF 决定了该层特征图捕获的信息范围。多尺度检测和分割中，不同层级的 RF 需要分别评估，不能只看最后一层。

4. **有 FC 层就忽略 RF 设计**：FC 层提供全局信息，但中间层的 RF 仍决定特征质量。在 FCN 时代的分割/检测中，没有 FC 兜底，RF 设计必须精确。

5. **理论 RF = 有效 RF**：有效 RF 通常只有理论值的 30%-50%。设计时需留出余量，尤其在需要精确上下文感知的任务中。有效 RF 可通过梯度加权可视化来测量。

6. **下采样次数越多越好**：RF 过大时，下采样虽然在数学上可行，但会急剧加速 RF 增长。需谨慎控制下采样策略——"网络最后面完全可以使用不下采样的空洞卷积来做"（`[[../cnn-cv/cnn-cv]]`）。
