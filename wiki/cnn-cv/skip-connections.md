# 跳跃连接

## 是什么

跳跃连接（Skip Connection）指神经网络中某一层的输出跨过若干层直接传递到更深层，使信号绕开中间变换直接传播。根据连接方式可分为三类主流形式：

**残差连接（Residual Connection / ResNet）**：通过恒等映射将输入加到输出上，`y = F(x) + x`。学习目标从"学习完整映射 F(x)"转变为"学习残差 `F(x) = y - x`"。当 F(x) 趋近于零时网络等价于恒等映射，深层网络至少不会比浅层差。ResNet 中每层实际学习的是"相对于恒等映射的偏离"，这一重新参数化（re-parameterization）大大降低了优化难度。残差连接要求输入输出维度一致，不一致时通过 zero-padding 或 1×1 卷积匹配。

**密集连接（Dense Connection / DenseNet）**：每一层的输入是前面所有层输出的拼接（concatenation），`x_l = H_l([x_0, x_1, ..., x_{l-1}])`。通过通道维度堆叠实现特征复用，避免 ResNet 求和可能带来的信息混淆。DenseNet 的增长率（growth rate）通常设为 32 或 48，控制每层新增的通道数。增长率越小意味着更好的参数复用，但需要更多层来达到相同的模型容量。显存占用随深度线性增长，但参数量远低于同等性能的 ResNet。

**捷径连接（Shortcut Connection）**：广义上指任何跨层连接。与残差连接不同，shortcut 不要求恒等映射——可以是 1×1 卷积、池化等变换。ResNet 提出前，Highway Networks（2015）已使用类似 LSTM 遗忘门的门控 shortcut，通过可学习的门控信号 `T(x)` 控制信息流：`y = F(x) × T(x) + x × (1 - T(x))`。Inception 系列中的并行分支也属于 shortcut 的广义范畴。

三者的共同直觉：让梯度在前向和反向传播中有更直接的路径，避免信息在多层非线性变换中衰减或丢失。`[[../cnn-cv/cnn-cv]]` 提到："shortcut connection 里，找不到 concat，用 add 凑合吧，反之亦然。" 虽然 add 和 concat 在数学上不等价，但在实践中可互相替代，体现了其核心价值在"提供捷径"而非具体运算。

## 为什么重要

跳跃连接是深度学习史上最重要的架构创新之一，直接解决了深层网络训练的核心障碍——退化问题（Degradation Problem）。

**解决梯度消失**：在反向传播中，梯度需从损失函数逐层回传。设网络第 l 层的损失梯度为 `∂L/∂x_l`，带残差连接的梯度为 `∂L/∂x_l = ∂L/∂x_{l+1} × (1 + ∂F/∂x_l)`。常数项 1 确保了即使 `∂F/∂x_l → 0`，梯度仍然可以完整回传。这就是恒等映射的核心价值——梯度通过恒等路径直接传播，不受中间层权重影响。对比普通网络 `∂L/∂x_l = ∂L/∂x_{l+1} × ∂F/∂x_l`，当所有层 `∂F/∂x_l < 1` 时梯度指数级衰减，`∂F/∂x_l > 1` 时指数级爆炸。残差连接打破了这种指数依赖。

**使超深网络成为可能**：ResNet 之前 20 层以上网络很难通过 SGD 优化——plain-56 的 top-1 错误率显著高于 plain-20，这就是退化现象。ResNet-152 在 ImageNet 上实现 top-5 错误率 3.57%，超越人类水平。ResNet-1001 在 CIFAR-10 上仍可收敛。没有残差连接，深层网络的训练误差会随深度增加而增大。`[[../cnn-cv/cnn-cv]]` 明确指出："深度决定了网络的表达能力，网络越深学习能力越强"——但只有残差连接才能让这种深度优势真正发挥。

**隐式集成（Implicit Ensemble）**：`[[../cnn-cv/cnn-cv]]` 指出："shortcut/inception/dense connection 都相当于隐式 ensemble。" 残差网络中信号可选择恒等路径或变换路径。Veit 等人的实验证明：移除 ResNet 中间某层后性能下降是平滑的而非灾难性的。`[[../cnn-cv/cnn-cv]]` 中强调："shortcut 几乎无痛提点"，正源于此。

**分割中保边界信息**：下采样丢失空间细节。U-Net 和 FPN 通过在编码器-解码器间添加跳跃连接，将浅层高分辨率特征传递到解码器，保留目标边界。`[[../cnn-cv/cnn-cv]]` 指出："在 dense prediction 领域……应该使用跳过连接，因为对象边界或有用的信息会由于最大池化操作或 strided convolutions 而丢失。"

**Pre-activation 与 Post-activation**：原始 ResNet 使用 Post-activation（Conv-BN-ReLU → 求和），Pre-activation ResNet（BN-ReLU-Conv → 求和）将激活放在卷积之前，使恒等路径完全"畅通"。在 1000 层以上训练更稳定，top-1 误差低约 0.5%。Kaiming He 在 ResNet v2 中分析了各种操作对恒等路径的影响——将 BN、ReLU、1×1 放在恒等路径上都会阻碍梯度流动。

**Stochastic Depth**：训练时以一定概率随机丢弃残差块（保留恒等路径），测试时使用全部块。这相当于 ResNet 的 Dropout——强迫每个残差块独立工作，防止过度依赖特定层。深度越深的网络使用随机深度训练的效果越好。

**WideResNet 的发现**：加宽比加深在某些数据集上更高效。WideResNet 将残差块的通道数增加（宽度因子 k），在深度不变的条件下大幅提升性能。这提示跳跃连接的关键贡献在于"让深网络可训练"，一旦可训练，宽度和深度需要共同调整。

**瓶颈设计（Bottleneck）**：ResNet bottleneck 通过 `1×1→3×3→1×1`（压缩→变换→扩展）大幅降低参数量。256 维输入先压缩到 64，3×3 在低维操作，再扩展回 256，计算量仅原版的 1/4。

## 如何使用

**残差连接**：网络超过 20 层时残差连接几乎是必要条件。10 层以下收益有限。分类/检测/分割 backbone 均适用。建议 Pre-activation 形式（BN-ReLU-Conv）。维度不匹配时用 1×1 conv（投影捷径）优于 zero-padding，约可多提 0.5 个点。

**密集连接**：特征复用在参数量较少时特别有效。DenseNet 适合数据量较小场景。显存占用随深度线性增长，过渡层（1×1 + 2×2 avg pool）做压缩和降采样。压缩因子通常取 0.5。高分辨率输入时 DenseNet 显存压力大，需权衡。

**快捷替换策略**：将现有模块替换为残差/密集连接时，`[[../cnn-cv/cnn-cv]]` 建议："通道数减半、密集连接只保留一半连接，做消融实验验证。" 降低风险，保留主要收益。

**ResNeXt 分组卷积扩展**：ResNeXt 将 3×3 替换为分组卷积，引入 cardinality 作为第三维度。相同 FLOPs 下增加 cardinality 比增加深度或宽度更有效。32×4d 的 ResNeXt-101 精度超 ResNet-152 但 FLOPs 更低。分组数建议设置为 32 或 64。

**分割/检测中的 Skip Connection**：编码器-解码器结构中（U-Net、FPN），连接从编码器中间层引出到解码器对应分辨率层。用 1×1 降维控制通道数。选择 3-4 层足够。多级连接（U-Net++ 的嵌套密集连接）可改善特征流动但增加计算量。

**ResNet vs DenseNet 选型**：

| 维度 | ResNet | DenseNet |
|------|--------|----------|
| 连接方式 | 求和（add） | 拼接（concat） |
| 参数效率 | 较低 | 较高（特征复用） |
| 显存占用 | 较低 | 较高（存中间图） |
| 训练难度 | 易 | 中等（增长率需调） |
| 特征复用 | 隐式（求和共享） | 显式（通道维串联） |
| 典型应用 | 通用 backbone | 小数据、分类 |

**与归一化的交互**：跳跃连接与 BN/LN 组合时，BN 可放加和前后。Pre-activation 在加和前，Post-activation 在加和后。Transformer 中的经验表明跳跃连接 + LN 是训练深 Transformer 的关键——但 LN 本身不解决梯度问题，LN 稳定了激活分布，跳跃连接提供了梯度路径，两者互补。

## 常见误区

1. **跳跃连接总是有益的**：不超过 10 层的浅网络收益可忽略，甚至可能增加计算。核心价值在超深网络中——解决退化而非提升表示能力。

2. **残差连接等价于 ensemble 所以效果好**：隐式 ensemble 仅在训练充分时有效。欠拟合时恒等路径偏好反而限制模型容量。残差连接是"让深网络可训练"的手段，不是"强网络更强"的万能药。

3. **密集连接优于残差连接**：DenseNet 参数效率高但显存增长是瓶颈。工程中 ResNet 及 ResNeXt 仍使用最广泛——稳定、易优化、推理快。

4. **拼接和求和完全等价**：求和要求维度一致限制灵活性；拼接保留所有信息但需 1×1 降维。工程上各有取舍。

5. **加跳跃连接就不需要 BatchNorm**：跳跃连接提供梯度路径但不解决激活分布偏移。BN 和跳跃连接互补——前者稳定训练，后者解决退化。

6. **投影捷径总是优于恒等捷径**：1×1 conv 对比 zero-padding 优势通常仅 0.5-1%，但引入额外参数。参数量敏感场景恒等捷径 + padding 足够。
