# Adam 优化器

## 是什么

Adam（Adaptive Moment Estimation）是2015年由Kingma和Ba提出的自适应学习率优化算法。它融合了两种方法的优点：

- **Momentum**：维护梯度的一阶矩（均值）估计，利用历史梯度方向平滑更新
- **RMSProp**：维护梯度的二阶矩（未中心化方差）估计，为每个参数自适应调整学习率

更新公式如下：

```
m_t = β1 * m_{t-1} + (1 - β1) * g_t
v_t = β2 * v_{t-1} + (1 - β2) * g_t^2
m̂_t = m_t / (1 - β1^t)
v̂_t = v_t / (1 - β2^t)
θ_t = θ_{t-1} - lr * m̂_t / (√v̂_t + ε)
```

其中 `g_t` 是当前梯度，`β1` 是一阶矩衰减率（通常0.9），`β2` 是二阶矩衰减率（通常0.999），`ε` 是数值稳定项（通常1e-8）。偏差修正（bias correction）确保训练初期估计值不会过小。

Adam的扩展变体包括：
- **AdamW**：将权重衰减从梯度更新中解耦，修正了原版Adam中L2正则化与自适应学习率的交互问题
- **Nadam**：融合Nesterov动量的Adam变体，收敛略快
- **Adamax**：基于L∞范数的Adam变体，对某些任务更稳定

## 为什么重要

Adam是当前深度学习实践中**最广泛使用的默认优化器**，原因如下：

1. **自适应学习率**：每个参数有自己的有效学习率，大幅减少手动调LR的工作量。稀疏特征对应的参数获得更大的更新步长，密集特征则步长更小——这对NLP、推荐系统等含大量稀疏特征的任务至关重要。

2. **超参数鲁棒**：默认超参数（β1=0.9, β2=0.999, ε=1e-8）在大多数任务上已经表现良好，不需要像SGD那样精细调节LR和动量。

3. **收敛速度快**：相比SGD，Adam在训练初期收敛显著更快，特别适合需要快速迭代实验的场景。

4. **处理非平稳目标**：对目标函数变化（如课程学习、GAN训练、RL）表现稳定。

但Adam有一个被广泛记录的问题：**泛化性能往往不如SGD+Momentum**。多项研究表明，自适应方法在训练集上的收敛速度更快，但在测试集上的泛化误差通常比精心调参的SGD大1-2个百分点。这可能是因为自适应学习率导致了"尖锐极小值"的收敛偏好。

## 如何使用

### 默认超参数

| 超参数 | 推荐值 | 说明 |
|--------|--------|------|
| 学习率 | 3e-4（通用首选） | 3e-4被广泛认为是Adam的最佳默认LR，比1e-3更稳定 |
| β1 | 0.9 | 一阶矩衰减率，越大动量越强 |
| β2 | 0.999 | 二阶矩衰减率，控制自适应步长的记忆长度 |
| ε | 1e-8 | 数值稳定项，避免除零 |

### 不同场景的LR参考

| 场景 | 推荐LR范围 | 说明 |
|------|-----------|------|
| 通用起始点 | 3e-4 | 原文载入的默认值 |
| CV任务 | 3e-4 ~ 5e-4 | 可配合Cosine衰减 |
| NLP Bert类 | 1e-5 ~ 3e-5 | 预训练模型微调更低 |
| Transformer从头训 | 1e-4 ~ 3e-4 | 配合Warmup必须 |
| LoRA微调 | 1e-4 ~ 3e-4 | 小数据需更保守 |
| PPO/RL | 1e-6 ~ 3e-6 | 比SFT小一个数量级 |
| DPO | 1e-7 ~ 5e-6 | 极保守，防止模式崩塌 |

### 使用建议

- **Adam几乎总是需要配合Warmup**，尤其是Transformer类模型。因为训练初期二阶矩估计不准确，直接使用大LR会导致参数更新过大。
- **推荐使用AdamW替代Adam**，尤其是在Transformer和微调场景下。AdamW修正了权重衰减的实现方式（见[[weight-decay]]）。
- **两阶段训练策略**：前期用Adam快速收敛，后期切换到SGD+Momentum提升泛化。这在追求极致效果时非常有效（参见[[../optimizer-lr#两阶段训练法Adam过渡到SGD]]）。
- **梯度裁剪**与Adam配合使用可以防止梯度爆炸导致的NaN，尤其在LLM微调中（参见[[../optimizer-lr#梯度裁剪防止NaN]]）。

## 常见误区

1. **认为Adam不需要调LR**。虽然Adam对LR比SGD鲁棒，但LR仍然是Adam最重要的超参数。3e-4只是起点，并非最优。

2. **混淆Adam和AdamW的权重衰减**。原版Adam的L2正则化与自适应LR交互会导致权重衰减的"有效强度"被自适应步长缩放，使得大梯度参数被正则化更多。AdamW解耦了这两个机制。见[[weight-decay#AdamW对Adam的改进]]。

3. **ε的重要性被忽视**。ε的默认值1e-8适用于float32训练。但在混合精度训练（float16）中，建议增大到1e-6或1e-7，否则小梯度平方项可能被精度截断导致训练不稳定。

4. **认为Adam一定比SGD好**。在CV领域，精心调参的SGD+Momentum泛化效果通常优于Adam。Adam的优势是"快速获得不错的结果"而非"最终效果最好"。

5. **β2设置过大**。β2=0.999在大多数场景工作良好，但如果训练步数较少（<10k步），较大的β2意味着二阶矩估计需要更多步数才能达到稳态，此时建议使用β2=0.99。
